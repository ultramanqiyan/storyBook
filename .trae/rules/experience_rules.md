# 经验查阅规则

## 规则概述

在需求、开发、测试、部署等各个环节出现问题和解决方案时，必须遵循以下流程：

## 流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    遇到问题/找到解决方案                      │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│          第一步：查阅 experience/ 目录下的文档                │
│  - experience/pitfalls/     踩坑经验                         │
│  - experience/solutions/    解决方案                         │
│  - experience/best-practices/ 最佳实践                       │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
              ┌───────────┴───────────┐
              │   是否找到相关经验？    │
              └───────────┬───────────┘
                    │           │
                   是           否
                    │           │
                    ▼           ▼
┌───────────────────────┐  ┌─────────────────────────────────┐
│  应用历史经验解决问题   │  │  使用苏格拉底式提问反思和总结    │
└───────────────────────┘  └───────────────┬─────────────────┘
                                           │
                                           ▼
                             ┌─────────────────────────────────┐
                             │  将新经验写入 experience/ 目录   │
                             └─────────────────────────────────┘
```

## 详细规则

### 第一步：查阅历史经验

在遇到问题时，必须先查阅以下目录：

| 目录 | 内容 | 适用场景 |
|------|------|----------|
| `experience/pitfalls/` | 历史踩坑记录 | 遇到Bug、错误、异常 |
| `experience/solutions/` | 历史解决方案 | 需要解决技术问题 |
| `experience/best-practices/` | 最佳实践 | 需要参考正确做法 |

### 第二步：判断是否有相关经验

查阅后判断是否存在相关或类似的历史经验：

- **有相关经验**：直接应用历史经验解决问题
- **无相关经验**：进入第三步

### 第三步：苏格拉底式提问反思

当没有找到相关历史经验时，使用苏格拉底式提问进行反思：

#### 提问模板

1. **问题定义**
   - 问题的现象是什么？
   - 预期的结果是什么？
   - 实际的结果是什么？

2. **原因分析**
   - 为什么会出现这个问题？
   - 问题的根本原因是什么？
   - 有哪些可能的因素导致？

3. **方案探索**
   - 有哪些可能的解决方案？
   - 每个方案的优缺点是什么？
   - 哪个方案最合适？为什么？

4. **预防措施**
   - 如何避免类似问题再次发生？
   - 需要添加什么规则或检查？
   - 有什么经验可以沉淀？

### 第四步：记录新经验

将反思结果写入对应的experience目录：

#### 写入规则

| 问题类型 | 写入目录 | 文件命名 |
|----------|----------|----------|
| Bug/错误 | `experience/pitfalls/` | `BUG-XXX-问题描述.md` |
| 技术方案 | `experience/solutions/` | `SOL-XXX-方案描述.md` |
| 最佳实践 | `experience/best-practices/` | `BP-XXX-实践描述.md` |

#### 文档模板

**踩坑经验模板** (`experience/pitfalls/BUG-XXX.md`):
```markdown
# [问题描述]

## 问题现象
[描述问题的具体表现]

## 根本原因
[分析问题的根本原因]

## 解决方案
[描述解决方案]

## 预防措施
[描述如何预防类似问题]

## 相关代码
[相关代码位置]

## 发现时间
[YYYY-MM-DD]

## 影响范围
[描述影响范围]
```

**解决方案模板** (`experience/solutions/SOL-XXX.md`):
```markdown
# [方案名称]

## 问题背景
[描述需要解决的问题背景]

## 方案描述
[详细描述解决方案]

## 实现步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 注意事项
[列出需要注意的事项]

## 相关资源
[相关文档或资源链接]
```

**最佳实践模板** (`experience/best-practices/BP-XXX.md`):
```markdown
# [实践名称]

## 适用场景
[描述适用的场景]

## 实践内容
[详细描述最佳实践内容]

## 示例代码
[提供示例代码]

## 注意事项
[列出需要注意的事项]

## 参考资料
[参考资料链接]
```

## 检查清单

在处理问题时，请确认：

- [ ] 是否已查阅 `experience/pitfalls/` 目录？
- [ ] 是否已查阅 `experience/solutions/` 目录？
- [ ] 是否已查阅 `experience/best-practices/` 目录？
- [ ] 如果是新问题，是否使用了苏格拉底式提问反思？
- [ ] 是否将新经验写入了对应的目录？
- [ ] 文档是否使用了正确的模板？

## 示例

### 示例1：遇到Bug

1. 遇到DOM空值访问错误
2. 查阅 `experience/pitfalls/` 发现 `BUG-003-DOM空值访问.md`
3. 应用历史经验：添加空值检查
4. 问题解决

### 示例2：新问题

1. 遇到新的CORS问题
2. 查阅 `experience/` 目录未找到相关经验
3. 使用苏格拉底式提问反思：
   - 为什么会出现CORS问题？
   - 有哪些解决方案？
   - 如何预防？
4. 将解决方案写入 `experience/solutions/SOL-001-CORS配置.md`
