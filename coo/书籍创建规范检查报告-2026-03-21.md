## 一、检查标准

根据《预设书籍制作指导文档》，每本书应具备以下标准目录结构：

```
coo/[book-name]/
├── chapters/                          # 章节内容目录
│   ├── chapter-01.md ~ chapter-10.md  # 10个章节文件
└── .progress/                         # 进度追踪目录
    ├── progress.json                  # 进度追踪文件
    ├── book-spec.md                   # 书籍规格文档
    ├── seo-meta.md                    # SEO元数据文档
    └── chapter-specs/                 # 章节规格目录
        ├── chapter-01-spec.md         # 第1章规格
        └── ... (chapter-10-spec.md)   # 共10个独立章节规格文件
```

### 必要文件清单

| 文件 | 必要性 | 说明 |
|------|--------|------|
| `.progress/book-spec.md` | ✅ 必要 | 书籍规格文档（全英文） |
| `.progress/progress.json` | ✅ 必要 | 进度追踪文件 |
| `.progress/seo-meta.md` | ✅ 必要 | SEO元数据文档 |
| `.progress/chapter-specs/` | ✅ 必要 | 章节规格目录 |
| `chapter-01-spec.md` ~ `chapter-10-spec.md` | ✅ 必要 | 10个独立章节规格文件 |
| `chapters/chapter-01.md` ~ `chapter-10.md` | ✅ 必要 | 10个章节内容文件 |

## 五、修复建议

### 5.1 高优先级修复方案

对于严重不符合规范的书籍，建议按以下步骤修复：

1. **创建目录结构**
   ```
   mkdir -p chapters
   mkdir -p .progress/chapter-specs
   ```

2. **创建 progress.json**
   ```json
   {
     "book": "[book-name]",
     "status": "pending",
     "current_chapter": 0,
     "total_chapters": 10,
     "started_at": "[YYYY-MM-DD]",
     "chapters": {}
   }
   ```

3. **创建 seo-meta.md** - 参考 `coo/.templates/` 中的模板

4. **创建章节规格文件** - 每章一个独立的 `chapter-XX-spec.md` 文件

5. **创建章节内容** - 每章一个 `chapter-XX.md` 文件

### 5.2 中优先级修复方案

对于章节规格使用合并文件的书籍：

1. 将合并文件 `chapters-01-10-specs.md` 拆分为 10 个独立文件
2. 每个文件命名为 `chapter-XX-spec.md`
3. 确保每个文件包含完整的章节规格内容

### 5.3 低优先级修复方案

对于章节规格混合格式的书籍：

1. 将合并文件中的内容拆分到独立文件
2. 确保所有 10 章都有独立的规格文件
