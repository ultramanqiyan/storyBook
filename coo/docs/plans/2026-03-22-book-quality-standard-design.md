> **文档版本**: 1.0
> **创建日期**: 2026-03-22

> **目标读者**: 内部技术团队

## 一、质检概述

### 1.1 质检目的

发布前质量把关，确保书籍在上线前达到质量标准，满足以下目标：
- 面向欧美读者，文化准确性有保障
- 通过Google搜索引擎导流，SEO优化到位
- AI痕迹最小化，避免被搜索引擎降权
- 内容质量过硬，读者体验良好

### 1.2 执行方式

| 项目 | 说明 |
|------|------|
| **执行主体** | AI自动质检 |
| **输出内容** | 质检报告 + 解决思路建议 |
| **禁止行为** | 不修改书籍内容 |
| **报告使用者** | 内部技术团队 |

### 1.3 质检范围

| 项目 | 数量 | 说明 |
|------|------|------|
| **书籍总数** | 41本 | coo目录下所有书籍，一本不遗漏 |
| **章节总数** | 约410章 | 每本书约10章 |
| **检测粒度** | 章节级 | 每个章节详细检测 |

### 1.4 执行要求（强制）

```
┌─────────────────────────────────────────────────────────────┐
│                    执行要求（强制）                           │
├─────────────────────────────────────────────────────────────┤
│  ✅ 必须串行执行：每本书、每章按顺序逐一检测                   │
│  ✅ 必须全面覆盖：41本书籍，一本不允许遗漏                     │
│  ✅ 必须详细检测：每个检查项逐一检测，一项不漏                  │
│  ✅ 必须一次执行完毕：不能分批次执行                           │
│  ❌ 严禁批量执行：不能使用批量处理方式                         │
│  ❌ 严禁并行执行：不能同时检测多本书或多章                     │
│  ❌ 严禁跳过检测：不能跳过任何检查项                           │
│  ❌ 严禁筛检执行：不能只检测部分内容                           │
│  ❌ 严禁快速执行：不能为了速度而降低检测质量                   │
└─────────────────────────────────────────────────────────────┘
```

## 二、质检维度与权重

### 2.1 维度总览

| 维度 | 权重 | 说明 | 优先级 |
|------|------|------|--------|
| 内容质量 | 20% | 情节连贯性、角色一致性、世界观一致性 | P0 |
| AI痕迹检测 | 15% | 十层检测框架，含Google E-E-A-T专项 | P0 |
| SEO优化 | 15% | 章节标题、关键词、元描述 | P0 |
| 欧美读者适配 | 15% | 文化准确性、生活场景真实性 | P0 |
| 语言连贯性 | 10% | 句子连贯、段落过渡、时态一致 | P1 |
| 技术实现 | 10% | 三层情感模型、感官细节、禁止元素 | P1 |
| 格式规范 | 5% | 章节标题格式、字数要求 | P2 |
| 法律风险 | 5% | 版权、敏感内容、商标侵权 | P2 |
| 读者体验 | 5% | 开篇吸引力、节奏控制、悬念设置 | P2 |
| 跨文档一致性 | - | 与规格文档、SEO文档、角色文档一致性 | P0 |

### 2.2 权重设计理由

| 维度 | 权重 | 理由 |
|------|------|------|
| 内容质量 | 20% | 核心价值，情节连贯性是基础 |
| AI痕迹检测 | 15% | Google会降权AI痕迹明显的内容 |
| SEO优化 | 15% | 直接影响搜索引擎导流效果 |
| 欧美读者适配 | 15% | 目标读者群体，文化准确性很重要 |
| 语言连贯性 | 10% | 阅读体验基础 |
| 技术实现 | 10% | 模板要求落实情况 |
| 格式规范 | 5% | 技术规范，易于修复 |
| 法律风险 | 5% | 风险控制，但发生概率低 |
| 读者体验 | 5% | 综合体验评估 |

## 三、评分机制

### 3.1 评分计算

```
总分 = Σ(维度得分 × 维度权重)

示例：
内容质量: 85分 × 20% = 17分
AI痕迹检测: 90分 × 15% = 13.5分
SEO优化: 80分 × 15% = 12分
欧美读者适配: 85分 × 15% = 12.75分
语言连贯性: 90分 × 10% = 9分
技术实现: 80分 × 10% = 8分
格式规范: 95分 × 5% = 4.75分
法律风险: 100分 × 5% = 5分
读者体验: 85分 × 5% = 4.25分
-----------------------------------
总分: 86.25分
```

### 3.2 发布门槛

| 条件 | 要求 | 说明 |
|------|------|------|
| 总分要求 | ≥ 70分 | 低于70分不可发布 |
| 严重问题 | 0个 | 有严重问题不可发布 |
| 中等问题 | ≤ 5个 | 中等问题过多需优化 |

### 3.3 质量等级

| 等级 | 分数范围 | 说明 | 发布建议 |
|------|----------|------|----------|
| A级 | 90-100分 | 优秀 | 可直接发布 |
| B级 | 80-89分 | 良好 | 可直接发布 |
| C级 | 70-79分 | 合格 | 建议优化后发布 |
| D级 | 60-69分 | 需改进 | 必须优化后发布 |
| F级 | <60分 | 不合格 | 建议重写 |

## 四、问题分级标准

### 4.1 分级定义

| 等级 | 定义 | 处理方式 |
|------|------|----------|
| **严重** | 必须修复才能发布，否则影响读者体验或SEO | 报告中高亮显示，优先处理 |
| **中等** | 建议修复后发布，影响部分读者体验 | 报告中标注，次优先处理 |
| **轻微** | 可选修复，影响较小 | 报告中列出，可选处理 |

### 4.2 分级示例

| 维度 | 严重 | 中等 | 轻微 |
|------|------|------|------|
| 内容质量 | 前后章节逻辑断裂 | 情节跳跃需补充 | 轻微过渡生硬 |
| AI痕迹 | 发现"tapestry/symphony"等AI词汇 | 发现轻微AI模式 | 个别陈词滥调 |
| SEO优化 | 标题无关键词 | 标题关键词不足 | 标题可优化 |

## 五、维度一：内容质量（权重20%）

### 5.1 检查项清单

| 检查项 | 严重 | 中等 | 轻微 |
|--------|------|------|------|
| 情节连贯性 | 前后章节逻辑断裂 | 情节跳跃需补充 | 轻微过渡生硬 |
| 角色一致性 | 角色设定前后矛盾 | 角色行为略显突兀 | 角色细节微调 |
| 世界观一致性 | 核心设定冲突 | 设定细节不一致 | 轻微设定模糊 |
| 章节衔接 | 章节间无逻辑联系 | 衔接略显生硬 | 可优化衔接 |
| 结局完整性 | 结局未完成/严重不合理 | 结局略显仓促 | 结局可更完善 |

### 5.2 评分规则

| 情况 | 得分 |
|------|------|
| 无问题 | 100分 |
| 1个轻微 | 95分 |
| 2个轻微 | 90分 |
| 1个中等 | 85分 |
| 1个中等 + 1个轻微 | 80分 |
| 2个中等 | 75分 |
| 1个严重 | 60分 |
| 1个严重 + 其他问题 | 50分 |
| 多个严重 | 40分 |

### 5.3 检测方法

```
┌─────────────────────────────────────────────────────────────┐
│                    内容质量检测流程                          │
├─────────────────────────────────────────────────────────────┤
│  1. 读取book-spec.md，提取核心设定                           │
│  2. 逐章检测：                                              │
│     - 主角姓名、年龄、职业是否一致                           │
│     - 故事主题是否一致                                       │
│     - 核心冲突是否一致                                       │
│     - 世界观设定是否一致                                     │
│  3. 检测章节衔接：                                          │
│     - 前后章节是否有逻辑连贯性                               │
│     - 时间线是否合理                                         │
│     - 事件发展是否自然                                       │
│  4. 检测结局完整性：                                        │
│     - 结局是否符合规格                                       │
│     - 结局是否合理                                           │
└─────────────────────────────────────────────────────────────┘
```
