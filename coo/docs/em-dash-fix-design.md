# 破折号批量替换设计文档

## 设计日期
2026-03-30

## 问题描述

coo目录下所有书籍章节存在大量破折号（—）过度使用的问题，这是AI写作的典型痕迹。

### 问题规模
- 书籍数量：104本
- 每书章节：约10章
- 总章节数：约1040章

### 问题模式
- 每段超过2个破折号
- 用破折号代替逗号或括号
- 用于戏剧性停顿

**示例**：
```
The first thing I noticed was the light—not light as I had ever known it—this was something entirely different.
```

## 解决方案

### 方法选择
**频率限制替换**：每段最多保留1个破折号，多余的替换为逗号。

### 理由
- 符合检测标准文档要求
- 不会完全破坏破折号的语义
- 实现相对简单，效果可控

## 实现设计

### 脚本位置
`coo/scripts/em_dash_fixer.py`

### 核心逻辑

```
1. 扫描 coo/*/chapters/*.md 所有章节文件
2. 按段落处理（空行分隔）
3. 每段统计破折号数量
4. 如果 > 1个：保留第一个，其余替换为逗号
5. 生成修改报告
```

### 替换规则

| 情况 | 处理方式 |
|------|----------|
| 段落有1个破折号 | 保留不变 |
| 段落有2+个破折号 | 保留第1个，其余替换为逗号 |
| 破折号前后有空格 | 替换为逗号+空格 |
| 破折号前后无空格 | 替换为逗号（无空格） |

### 输出报告

保存到 `coo/docs/em-dash-fix-report.md`，包含：
- 处理文件总数
- 修改总数
- 每个文件的修改详情（行号、原文、修改后）

### 安全措施

- 修改前自动备份原文件到 `coo/backup/` 目录
- 报告中记录所有修改，便于回溯

## 预期结果

- 所有章节文件破折号使用符合标准
- 每段最多保留1个破折号
- 生成完整的修改报告
- 原文件有备份保护

## 风险评估

| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| 误替换正确的破折号 | 低 | 频率限制保留1个 |
| 文件损坏 | 低 | 自动备份机制 |
| 编码问题 | 低 | 使用UTF-8编码 |
