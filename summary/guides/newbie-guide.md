## 第一章：项目概览

### 1.1 项目是什么

卡牌互动小说网站是一个让用户通过收集卡牌、选择角色和情节来创作和阅读互动小说故事的在线平台。

**核心玩法**：

| 玩法 | 说明 |
|------|------|
| 创建书籍 | 选择书籍类型（儿童冒险/魔幻传说/都市言情/职场风云） |
| 创建角色 | 设计主角和配角，设定性格和关系 |
| 收集卡牌 | 通过解谜获得天气、地形、装备等卡牌 |
| 生成章节 | 选择卡牌组合，AI辅助生成故事章节 |
| 解谜互动 | 每个章节包含谜题，增加互动性 |

**目标用户**：10-40岁，喜欢故事创作和卡牌收集的用户，主要面向美国、欧洲等海外市场。

### 1.4 核心概念解释

| 概念 | 说明 |
|------|------|
| **书籍(Book)** | 一个互动小说项目，包含角色、卡牌、章节 |
| **角色(Character)** | 书籍中的角色，包括主角和配角 |
| **卡牌(Card)** | 情节元素，包括角色卡牌、天气卡牌、地形卡牌等 |
| **章节(Chapter)** | 故事的一个章节，由卡牌组合生成 |
| **谜题(Puzzle)** | 每个章节包含的互动谜题 |
| **预设书籍** | 系统预设的书籍，未登录用户可查看 |
| **用户书籍** | 用户创建的书籍，需要登录 |

### 本章小结

**核心要点**：
1. 项目是一个卡牌互动小说网站
2. 技术栈：Cloudflare全家桶 + 原生前端
3. 核心功能：用户系统、书籍系统、卡牌系统、章节系统、谜题系统

**关键记忆点**：
- 4种书籍类型：儿童冒险、魔幻传说、都市言情、职场风云
- 5种卡牌类型：角色、天气、地形、冒险类型、装备
- 数据隔离：所有查询必须包含user_id

### 5.3 API响应格式

**统一格式**：

```json
{
  "success": true,
  "data": { ... },
  "error": null
}
```

**错误响应**：

```json
{
  "success": false,
  "data": null,
  "error": "错误信息"
}
```

**HTTP状态码**：

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 5.4 数据库操作规范

**基本原则**：
1. 所有查询必须包含user_id过滤（数据隔离）
2. 使用参数化查询防止SQL注入
3. 大表查询必须使用索引
4. 批量操作使用事务

**示例**：

```javascript
// 正确：包含user_id过滤
const result = await env.DB.prepare(
  'SELECT * FROM books WHERE user_id = ?'
).bind(userId).all();

// 错误：缺少user_id过滤
const result = await env.DB.prepare(
  'SELECT * FROM books'
).all();
```

### 5.5 安全注意事项

**检查清单**：

| 项目 | 说明 |
|------|------|
| 数据隔离 | 所有查询包含user_id |
| 参数校验 | 验证所有输入参数 |
| SQL注入 | 使用参数化查询 |
| XSS防护 | 转义用户输入 |
| 密码加密 | 使用bcrypt |
| 敏感信息 | 不提交到代码库 |

### 本章小结

**核心要点**：
1. 命名规范：camelCase变量，snake_case数据库字段
2. Git提交：使用规范格式
3. API响应：统一格式 {success, data, error}
4. 数据库：必须包含user_id过滤
5. 安全：数据隔离、参数校验、SQL注入防护

**关键记忆点**：
- 数据隔离是最重要的安全措施
- 所有查询必须包含user_id

## 第八章：常见任务指南

### 8.1 如何新增一个API接口

**步骤**：

1. 在 `functions/api/` 目录下创建文件
2. 遵循API响应格式规范
3. 添加参数校验和错误处理
4. 编写单元测试
5. 更新API文档

**示例**：

```javascript
// functions/api/example.js
export async function onRequestGet(context) {
  const { env, request } = context;
  
  try {
    // 1. 参数校验
    const url = new URL(request.url);
    const id = url.searchParams.get('id');
    
    if (!id) {
      return Response.json({
        success: false,
        data: null,
        error: '缺少id参数'
      }, { status: 400 });
    }
    
    // 2. 数据库查询
    const result = await env.DB.prepare(
      'SELECT * FROM example WHERE id = ?'
    ).bind(id).first();
    
    // 3. 返回结果
    return Response.json({
      success: true,
      data: result,
      error: null
    });
    
  } catch (error) {
    return Response.json({
      success: false,
      data: null,
      error: error.message
    }, { status: 500 });
  }
}
```

);

CREATE INDEX IF NOT EXISTS idx_example_user_id ON example(user_id);
