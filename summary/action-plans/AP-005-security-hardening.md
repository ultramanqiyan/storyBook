## 二、会议讨论

### 议题1：请求限流

**PM-老张**：请求限流怎么做？

**后端-小刘**：**方案一：IP限流**。限制每个IP的请求频率。优点是简单，缺点是可能误伤（多人共用IP）。

**后端-小刘**：**方案二：用户限流**。限制每个用户的请求频率。优点是精准，缺点是需要用户登录。

**后端-小刘**：**方案三：混合限流**。未登录用户IP限流，登录用户用户限流。优点是平衡，缺点是复杂。

**PM-老张**：方案三。具体限制怎么定？

**后端-小刘**：
- 未登录用户：每IP每分钟20次请求
- 登录用户：每用户每分钟100次请求
- AI生成接口：每用户每天10次（已在成本控制方案中）

**PM-老张**：超限怎么处理？

**后端-小刘**：返回429状态码，提示"请求过于频繁，请稍后再试"。

**【决议】**
- 未登录用户：IP限流20次/分钟
- 登录用户：用户限流100次/分钟
- AI接口：单独限流（成本控制方案）

### 议题2：XSS防护

**PM-老张**：XSS防护怎么做？

**前端-小陈**：**方案一：输入过滤**。过滤用户输入的特殊字符。优点是源头处理，缺点是可能过滤正常内容。

**前端-小陈**：**方案二：输出转义**。输出时转义特殊字符。优点是安全，缺点是每个输出点都要处理。

**前端-小陈**：**方案三：CSP策略**。设置内容安全策略。优点是防护强，缺点是配置复杂。

**PM-老张**：方案一+方案二+方案三组合。输入过滤+输出转义+CSP策略。

**前端-小陈**：CSP策略怎么配置？

**运维-老孙**：在HTTP响应头中设置Content-Security-Policy。

**【决议】**
- 输入过滤特殊字符
- 输出转义特殊字符
- 配置CSP策略

### 议题3：CSRF防护

**PM-老张**：CSRF防护怎么做？

**后端-小刘**：**方案一：CSRF Token**。每个请求携带Token验证。优点是安全，缺点是实现复杂。

**后端-小刘**：**方案二：SameSite Cookie**。设置Cookie的SameSite属性。优点是简单，缺点是兼容性问题。

**后端-小刘**：**方案三：双重验证**。Token+SameSite组合。优点是双重保护，缺点是复杂。

**PM-老张**：方案一+方案二组合。CSRF Token+SameSite Cookie。

**【决议】**
- 实现CSRF Token验证
- 设置Cookie SameSite属性

### 议题4：账号找回

**PM-老张**：账号找回怎么做？

**后端-小刘**：**方案一：邮箱验证**。发送重置链接到邮箱。优点是标准，缺点是需要邮件服务。

**后端-小刘**：**方案二：安全问题**。设置安全问题验证。优点是简单，缺点是不够安全。

**后端-小刘**：**方案三：管理员重置**。联系管理员手动重置。优点是简单，缺点是不专业。

**PM-老张**：方案一。邮箱验证是标准做法。

**后端-小刘**：需要邮件服务，有成本。

**PM-老张**：用第三方邮件服务，比如SendGrid免费版。

**【决议】**
- 实现邮箱验证找回密码
- 使用第三方邮件服务

## 四、具体执行方案

### 4.1 请求限流实施

#### 4.1.1 限流规则

| 用户类型 | 限流维度 | 限制 | 超限处理 |
|----------|----------|------|----------|
| 未登录用户 | IP | 20次/分钟 | 返回429 |
| 登录用户 | 用户ID | 100次/分钟 | 返回429 |
| AI生成接口 | 用户ID | 10次/天 | 返回429 |

#### 4.1.2 技术实现

**实现方式**：使用Cloudflare Workers KV存储请求计数

**限流算法**：滑动窗口算法

**代码逻辑**：
```
1. 获取限流Key（IP或用户ID）
2. 从KV获取当前计数
3. 判断是否超限
4. 未超限：计数+1，继续请求
5. 超限：返回429
```

### 4.2 XSS防护实施

#### 4.2.1 输入过滤

**过滤规则**：

| 字符 | 过滤为 | 说明 |
|------|--------|------|
| < | &lt; | 小于号 |
| > | &gt; | 大于号 |
| " | &quot; | 双引号 |
| ' | &#x27; | 单引号 |
| / | &#x2F; | 斜杠 |

**过滤位置**：
- 用户输入的文本内容
- URL参数
- 表单数据

#### 4.2.2 输出转义

**转义函数**：
```javascript
function escapeHtml(text) {
  const map = {
    '&': '&amp;',
    '<': '&lt;',
    '>': '&gt;',
    '"': '&quot;',
    "'": '&#039;'
  };
  return text.replace(/[&<>"']/g, m => map[m]);
}
```

#### 4.2.3 CSP策略

**CSP配置**：
```
Content-Security-Policy: 
  default-src 'self';
  script-src 'self' 'unsafe-inline';
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: https:;
  connect-src 'self' https://api.doubao.com;
```

### 4.3 CSRF防护实施

#### 4.3.1 CSRF Token

**Token生成**：
- 用户登录时生成随机Token
- 存储在localStorage
- 每次请求携带Token

**Token验证**：
- 服务器验证Token有效性
- Token过期时间：24小时

#### 4.3.2 SameSite Cookie

**Cookie配置**：
```
Set-Cookie: session=xxx; HttpOnly; Secure; SameSite=Strict
```

### 4.4 账号找回实施

#### 4.4.1 找回流程

```
用户点击"忘记密码"
    ↓
输入注册邮箱
    ↓
发送重置链接到邮箱
    ↓
用户点击链接
    ↓
验证链接有效性
    ↓
设置新密码
    ↓
密码重置成功
```

#### 4.4.2 技术实现

**重置链接**：
- 格式：`/reset-password?token=xxx`
- Token：随机字符串，有效期24小时
- 存储：数据库中存储Token和过期时间

**邮件服务**：
- 使用SendGrid免费版
- 每月100封免费
