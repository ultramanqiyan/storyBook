## 一、问题背景

### 1.1 问题现象

**后端-小刘**：我们的AI服务（豆包API）有时候会超时，用户体验很差。

**运维-老孙**：而且我们没有任何降级策略，AI一挂，核心功能就废了。

**产品-小王**：语音服务（SiliconFlow）我们预留了但一直没用，要不要评估下？

**PM-老张**：外部服务是项目的命脉，必须建立完整的管理机制。

### 1.2 核心问题

| 问题维度 | 具体表现 | 影响程度 |
|---------|---------|---------|
| AI服务不稳定 | 豆包API偶发超时 | 核心功能受影响 |
| 无降级策略 | 服务故障时无备选方案 | 用户体验崩溃 |
| 语音服务闲置 | SiliconFlow预留未用 | 资源浪费 |
| 选型不规范 | 缺乏系统评估流程 | 可能选错服务商 |
| 成本不可控 | 无用量监控和预警 | 成本风险 |

## 二、会议讨论

### 2.1 AI服务稳定性

**后端-小刘**：豆包API是我们最核心的依赖，但稳定性确实有问题。

**运维-老孙**：我统计了一下，最近一周有3次超时，每次影响几十个用户。

**产品-小王**：用户反馈说"生成失败"很影响体验，有人因此流失了。

#### 方案对比

| 方案 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| 方案A：增加重试机制 | 简单有效 | 可能增加延迟 | 低 |
| 方案B：多服务商备份 | 高可用 | 成本翻倍、切换复杂 | 高 |
| 方案C：本地缓存+降级 | 成本低、体验好 | 内容可能过时 | 低 |

**决议**：采用方案A+C组合，重试+缓存+降级。

### 2.2 语音服务评估

**产品-小王**：SiliconFlow的语音服务我们预留了接口，但一直没启用。

**后端-小刘**：当时考虑的是让故事可以朗读出来，增加沉浸感。

**PM-老张**：这个功能用户需要吗？投入产出比如何？

#### 方案对比

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| 方案A：立即启用 | 功能完整 | 成本高、需求不明确 | 不推荐 |
| 方案B：用户调研后决定 | 数据驱动 | 延迟上线 | 推荐 |
| 方案C：暂不启用 | 节省成本 | 功能缺失 | 可选 |

**决议**：采用方案B，先做用户调研，根据需求决定。

### 2.3 第三方服务选型

**PMO-小李**：我们选外部服务太随意了，没有标准流程。

**运维-老孙**：确实，豆包API是临时决定的，没有对比其他选项。

**后端-小刘**：需要建立选型评估流程，避免以后再犯同样的错误。

#### 选型标准

| 维度 | 权重 | 评估项 |
|------|------|--------|
| 稳定性 | 30% | SLA、故障历史、恢复时间 |
| 成本 | 25% | 单价、阶梯定价、隐藏成本 |
| 功能 | 20% | 功能完整性、API友好度 |
| 支持 | 15% | 文档质量、响应速度 |
| 合规 | 10% | 数据安全、隐私合规 |

**决议**：建立选型评估表，所有新服务必须评估。

### 2.4 服务降级策略

**后端-小刘**：当外部服务不可用时，我们需要有降级方案。

**产品-小王**：降级后用户体验会差一些，但至少能用。

**运维-老孙**：需要定义降级触发条件和恢复条件。

#### 降级策略

| 服务 | 降级方案 | 触发条件 | 恢复条件 |
|------|---------|---------|---------|
| AI生成 | 返回预设模板 | 连续失败3次 | 服务恢复+测试通过 |
| AI生成 | 缓存历史内容 | 响应超时 | 服务恢复 |
| 语音服务 | 关闭语音功能 | 服务不可用 | 服务恢复 |

**决议**：实现分级降级策略，确保核心功能可用。

#### 4.1.1 豆包API配置

| 配置项 | 当前值 | 优化值 | 说明 |

| 超时时间 | 30秒 | 60秒 | 增加容错 |

| 并发限制 | 无 | 10 | 防止过载 |

#### 4.1.2 重试机制实现

```javascript
const retryConfig = {
  maxRetries: 3,
  baseDelay: 1000,
  maxDelay: 10000,
  retryableErrors: ['ETIMEDOUT', 'ECONNRESET', 'ENOTFOUND']
};

async function callAIWithRetry(prompt, config = retryConfig) {
  let lastError;
  for (let i = 0; i <= config.maxRetries; i++) {
    try {
      return await callDoubaoAPI(prompt);
    } catch (error) {
      lastError = error;
      if (!config.retryableErrors.includes(error.code)) {
        throw error;
      }
      if (i < config.maxRetries) {
        const delay = Math.min(config.baseDelay * Math.pow(2, i), config.maxDelay);
        await sleep(delay);
      }
    }
  }
  throw lastError;
}
```

## 七、风险与应对

### 7.1 风险清单

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|---------|

### 7.2 应急预案

```
AI服务故障应急预案：
1. 服务不可用（<5分钟）
   └── 自动重试
   └── 启用缓存

2. 服务不可用（5-30分钟）
   └── 启用降级模式
   └── 通知用户

3. 服务不可用（>30分钟）
   └── 切换备用服务商

   └── 联系服务商
```
