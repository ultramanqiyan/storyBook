## 一、问题背景

### 1.1 问题现象

**产品-小王**：我们的用户增长完全依赖自然流量，太慢了。

**运营-老孙**：而且没有用户推荐机制，用户没有动力帮我们拉新。

**PM-老张**：邀请奖励是最低成本的增长方式，必须做。

**后端-小刘**：但是要防止刷奖励，需要设计好风控机制。

### 1.2 核心问题

| 问题维度 | 具体表现 | 影响程度 |
|---------|---------|---------|
| 增长渠道单一 | 仅靠自然搜索和社媒 | 增长缓慢 |
| 无推荐激励 | 用户无动力拉新 | 错失免费流量 |
| 无病毒传播 | 缺乏传播机制 | 用户增长受限 |
| 无裂变机制 | 无老带新机制 | 用户规模难扩大 |

## 二、会议讨论

### 2.1 邀请奖励机制

**产品-小王**：邀请奖励怎么设计？

**PM-老张**：要平衡激励效果和成本。

**后端-小刘**：**方案一：固定奖励**。每邀请1人奖励X张卡牌。优点是简单，缺点是容易被刷。

**后端-小刘**：**方案二：阶梯奖励**。邀请越多奖励越多。优点是激励强，缺点是计算复杂。

**后端-小刘**：**方案三：双向奖励**。邀请人和被邀请人都有奖励。优点是转化率高，缺点是成本高。

**PM-老张**：采用方案三，双向奖励+阶梯奖励组合。

#### 方案对比

| 方案 | 激励效果 | 成本 | 风控难度 | 推荐 |
|------|---------|------|---------|------|
| 固定奖励 | 低 | 低 | 低 | 不推荐 |
| 阶梯奖励 | 中 | 中 | 中 | 可选 |
| 双向奖励 | 高 | 高 | 中 | 推荐 |
| 组合方案 | 高 | 中 | 中 | 推荐 |

**决议**：采用双向奖励+阶梯奖励组合方案。

### 2.2 奖励内容设计

**产品-小王**：奖励什么？

**运营-老孙**：卡牌是最核心的资源，用户最想要。

**PM-老张**：也可以奖励AI生成次数，这是成本项。

#### 奖励类型

| 奖励类型 | 成本 | 吸引力 | 可控性 |
|---------|------|--------|--------|
| 稀有卡牌 | 低 | 高 | 高 |
| AI生成次数 | 中 | 中 | 中 |
| 专属皮肤 | 低 | 中 | 高 |
| 会员天数 | 高 | 高 | 低 |

**决议**：主要奖励稀有卡牌，辅以AI生成次数。

### 2.3 风控机制

**后端-小刘**：邀请奖励最大的风险是刷奖励。

**PM-老张**：怎么防？

**后端-小刘**：**方案一：被邀请人需完成新手任务**。邀请人才能获得奖励。优点是有效防刷，缺点是门槛高。

**后端-小刘**：**方案二：延迟发放**。奖励延迟X天发放。优点是可追溯，缺点是体验差。

**后端-小刘**：**方案三：行为检测**。检测异常邀请行为。优点是自动化，缺点是可能误判。

**PM-老张**：方案一+方案三组合。

#### 风控规则

| 规则 | 说明 | 触发条件 |
|------|------|---------|
| 新手任务 | 被邀请人完成3个新手任务 | 强制 |
| 设备检测 | 同一设备只能注册一次 | 强制 |
| 行为检测 | 检测异常邀请频率 | 触发审核 |
| 人工审核 | 高价值奖励人工审核 | 邀请>10人 |

**决议**：实施多层风控机制。

### 2.4 病毒传播机制

**产品-小王**：除了邀请奖励，还有什么传播方式？

**运营-老孙**：可以分享故事、分享卡牌。

**前端-小陈**：分享时带上用户标识，别人通过分享链接注册也算邀请。

#### 传播场景

| 场景 | 触发时机 | 分享内容 | 奖励 |
|------|---------|---------|------|
| 完成故事 | 故事完结时 | 故事链接 | 1张卡牌 |
| 获得稀有卡牌 | 获得时 | 卡牌图片 | 无 |
| 达成成就 | 达成时 | 成就卡片 | 无 |
| 邀请好友 | 主动邀请 | 邀请链接 | 阶梯奖励 |

**决议**：实现多场景分享传播。

## 四、具体执行方案

### 4.1 邀请机制

#### 4.1.1 邀请码设计

| 项目 | 设计 |
|------|------|
| 格式 | 8位字母数字组合（如：STORY1A2B） |
| 生成规则 | 用户注册时自动生成，全局唯一 |
| 有效期 | 永久有效 |
| 使用限制 | 每个邀请码可被无限人使用 |

#### 4.1.2 邀请链接设计

```
https://storybook.app/invite/STORY1A2B
https://storybook.app/?ref=STORY1A2B
```

#### 4.1.3 邀请记录

| 字段 | 说明 |
|------|------|
| inviter_id | 邀请人ID |
| invitee_id | 被邀请人ID |
| invite_code | 邀请码 |
| invite_time | 邀请时间 |
| reward_status | 奖励状态 |
| reward_time | 奖励发放时间 |

### 4.2 奖励机制

#### 4.2.1 双向奖励

| 角色 | 奖励 | 条件 |
|------|------|------|
| 邀请人 | 2张稀有卡牌 | 被邀请人完成3个新手任务 |
| 被邀请人 | 1张稀有卡牌 | 通过邀请链接注册 |

#### 4.2.2 阶梯奖励

| 邀请人数 | 额外奖励 |
|---------|---------|
| 5人 | 5张AI生成券 |
| 10人 | 10张AI生成券 + 专属头像框 |
| 20人 | 20张AI生成券 + 专属称号 |
| 50人 | 50张AI生成券 + 1个月会员 |
| 100人 | 100张AI生成券 + 3个月会员 |

#### 4.2.3 奖励发放流程

```
奖励发放流程：
┌─────────────┐
│ 被邀请人注册 │
└──────┬──────┘
       │
       ▼
┌─────────────┐     完成     ┌─────────────┐
│ 完成新手任务 │────────────▶│ 发放奖励    │
└──────┬──────┘              └─────────────┘
       │ 未完成
       ▼
┌─────────────┐
│ 7天后过期   │
└─────────────┘
```

### 4.3 风控机制

#### 4.3.1 新手任务要求

| 任务 | 说明 | 目的 |
|------|------|------|
| 创建书籍 | 创建第一本书籍 | 验证真实使用 |
| 创建角色 | 创建第一个角色 | 验证真实使用 |
| 生成章节 | 生成第一个章节 | 验证真实使用 |

#### 4.3.2 设备检测

| 检测项 | 说明 | 处理 |
|--------|------|------|
| 设备ID | 检测设备唯一标识 | 同设备只能注册一次 |
| IP地址 | 检测注册IP | 同IP短时间多次注册触发审核 |
| 浏览器指纹 | 检测浏览器特征 | 异常指纹触发审核 |

#### 4.3.3 行为检测规则

| 规则 | 阈值 | 处理 |
|------|------|------|
| 单日邀请数 | >10人 | 人工审核 |
| 邀请转化率 | <10% | 标记异常 |
| 被邀请人活跃度 | <1分钟 | 标记异常 |
| 同IP邀请数 | >3人 | 人工审核 |

### 4.4 传播场景

#### 4.4.1 故事分享

| 场景 | 分享内容 | 触发时机 |

```javascript
function generateInviteCode(userId) {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
  const timestamp = Date.now().toString(36);
  const hash = simpleHash(userId + timestamp);
  let code = '';
  for (let i = 0; i < 8; i++) {
    code += chars[(hash + i) % chars.length];
  }
  return code;
}
```
