# 移动端Web UI测试设计方案

## 概述

### 测试目标

测试网站在移动端的表现，重点关注：
- 展示问题（遮挡、看不清、丑陋、不合理）
- 布局问题（卡牌展示异常、章节内容太小、书籍页丑陋、导演页奇怪）
- 配色问题
- 交互问题

### 测试范围

| 维度 | 内容 |
|-----|------|
| **设备** | 中屏手机(390px)、大屏手机(428px)、平板(768px) |
| **浏览器** | Safari(iOS)、Chrome(Android) |
| **语言** | 英文（面向欧美用户） |
| **页面** | 书架、书籍详情、章节阅读、导演页、创建书籍 |

### 测试环境

- **本地环境**：`http://127.0.0.1:8788`
- **数据库**：本地SQLite
- **浏览器**：Chromium（移动端模拟）

## 检查清单

### 布局与尺寸

- [ ] 安全区域适配（刘海屏/灵动岛/底部手势区）
- [ ] 元素不超出屏幕边界
- [ ] 无意外横向滚动
- [ ] 触摸目标≥44px
- [ ] 单手操作可达性（重要按钮在下半屏）

### 可读性

- [ ] 正文≥14px（儿童建议≥16px）
- [ ] 标题≥18px
- [ ] 行高1.5-1.8
- [ ] 对比度≥4.5:1
- [ ] 长文本正确截断/换行
- [ ] 色彩盲友好（状态不只靠颜色区分）

### 交互体验

- [ ] 触摸反馈明显（按下态）
- [ ] 焦点状态清晰
- [ ] 禁用状态明显
- [ ] 键盘弹出不遮挡输入框
- [ ] 加载状态有反馈
- [ ] 防误触（删除等危险操作有确认）

### 状态处理

- [ ] 空状态有友好提示
- [ ] 错误状态有清晰说明
- [ ] 加载状态有进度指示
- [ ] 网络异常有处理
- [ ] 会话超时有处理

### 弹窗与模态框

- [ ] 弹窗不超出屏幕
- [ ] 关闭按钮可点击
- [ ] 背景滚动锁定
- [ ] 键盘弹出时弹窗上移

### 动画与效果

- [ ] 卡牌翻转动画正常
- [ ] 粒子效果不影响性能
- [ ] 滚动流畅无卡顿
- [ ] 动画不影响可点击性

### 特殊场景

- [ ] 横屏适配正常
- [ ] 动态字体适配
- [ ] 暗黑模式对比度
- [ ] 多语言布局适配
- [ ] 分屏模式适配
- [ ] 强光下可读性

### 浏览器兼容

- [ ] Safari渲染正常
- [ ] 微信内置浏览器正常
- [ ] CSS前缀完整
- [ ] Touch事件正常

### 可访问性（WCAG 2.1 AA）

- [ ] 对比度≥4.5:1（普通文本）
- [ ] 对比度≥3:1（大文本）
- [ ] 焦点状态清晰可见
- [ ] 所有图片有alt属性
- [ ] 所有表单有label
- [ ] 错误信息清晰描述
- [ ] 可通过键盘完成所有操作
- [ ] 动画可暂停

### 欧美市场专项

- [ ] 英文文本不溢出容器
- [ ] 长单词正确换行
- [ ] 日期格式符合当地习惯
- [ ] Cookie同意弹窗存在（如需要）
- [ ] 隐私政策链接可见

## 测试用例设计

### 01-static-analysis.spec.js - 静态CSS分析

不启动浏览器，分析CSS文件：

| 检查项 | 说明 |
|-------|------|
| 响应式断点 | 检查是否存在 768px、480px 等断点 |
| 固定宽度元素 | 检查是否有超出移动端宽度的固定宽度 |
| 字体大小 | 检查移动端字体是否小于12px |
| 触摸目标 | 检查按钮/链接是否足够大（最小44px） |
| 横向滚动 | 检查是否有可能导致横向滚动的元素 |

### 02-display-pages.spec.js - 页面展示测试

| 测试场景 | 检查项 |
|---------|-------|
| **书架页面** | 书籍卡片是否正常显示、网格布局是否合理、导航栏是否遮挡内容 |
| **书籍详情页** | 书籍信息是否清晰、章节列表是否可滚动、角色/剧情卡片是否正常 |
| **章节阅读页** | 文字是否可读、翻页按钮是否可点击、谜题弹窗是否正常显示 |
| **导演页** | 三栏布局是否变为单栏、卡牌选择是否正常、舞台预览是否可见 |

### 03-book-creation.spec.js - 创建书籍流程

| 步骤 | 检查项 |
|-----|-------|
| 1. 登录 | 表单是否可填写、提交按钮是否可点击 |
| 2. 进入创建页 | 页面是否正常加载 |
| 3. 填写书籍信息 | 输入框是否可聚焦、下拉选择是否正常 |
| 4. 选择剧情卡 | 卡片是否可点击、选中状态是否明显 |
| 5. 选择角色 | 角色列表是否可滚动、选择是否正常 |
| 6. 确认创建 | 提交按钮是否可点击、等待响应是否正常 |
| 7. 跳转详情页 | 是否成功跳转、数据是否正确显示 |

### 04-chapter-creation.spec.js - 添加章节流程

| 步骤 | 检查项 |
|-----|-------|
| 1. 进入书籍详情 | 章节列表是否显示 |
| 2. 点击添加章节 | 按钮是否可点击 |
| 3. 选择角色 | 角色选择弹窗是否正常显示、复选框是否可选 |
| 4. 选择剧情 | 剧情选项是否可点击 |
| 5. 确认生成 | 提交后是否显示加载状态 |
| 6. 查看新章节 | 新章节是否出现在列表中 |

### 05-director-flow.spec.js - 导演页流程

| 步骤 | 检查项 |
|-----|-------|
| 1. 进入导演页 | 页面布局是否适配移动端 |
| 2. 选择主角 | 卡牌扇形展开是否正常、选中状态是否明显 |
| 3. 选择配角 | 是否可选多个、数量限制是否生效 |
| 4. 选择剧情卡 | 各类型卡牌是否可点击 |
| 5. 舞台预览 | 选中卡牌是否正确显示在舞台位 |
| 6. 开始制作 | 按钮是否可点击、加载状态是否正常 |
| 7. 跳转章节 | 是否成功跳转到新章节 |

### 06-edge-cases.spec.js - 边缘场景测试

| 测试场景 | 检查项 |
|---------|-------|
| 网络断开 | 断网时的提示和处理 |
| 弱网环境 | 加载超时的处理 |
| 长文本 | 标题/描述过长时的截断或换行 |
| 空数据 | 无数据时的占位显示 |
| 键盘弹出 | 输入框是否被键盘遮挡 |
| 横屏模式 | 横屏时布局是否正常 |

### 07-accessibility.spec.js - 可访问性测试

| 测试场景 | 检查项 |
|---------|-------|
| 对比度检查 | 文字与背景对比度是否符合WCAG标准 |
| 焦点可见 | 键盘导航时焦点是否可见 |
| 替代文本 | 图片是否有alt属性 |
| 表单标签 | 表单元素是否有label |
| 键盘导航 | 是否可以通过键盘完成所有操作 |

### 08-localization.spec.js - 本地化测试

| 测试场景 | 检查项 |
|---------|-------|
| 英文文本 | 文本是否溢出容器 |
| 长单词 | 是否正确换行 |
| 日期格式 | 是否符合当地习惯 |
| 数字格式 | 是否正确显示 |

## 运行方式

```bash
# 终端1：启动开发服务器
npm run dev

# 终端2：运行移动端测试（headed模式，可以看到浏览器）
npx playwright test tests/e2e/mobile --project=medium-phone --headed --reporter=list

# 运行所有设备测试
npx playwright test tests/e2e/mobile --headed --reporter=list

# 运行单个测试文件
npx playwright test tests/e2e/mobile/specs/02-display-pages.spec.js --headed
```

## 历史经验参考

| 问题 | 解决方案 |
|------|---------|
| Playwright测试卡住 | 不使用webServer自动启动，手动启动开发服务器后再运行测试 |
| 书籍创建流程超时 | 先等待API响应，使用更长超时时间，添加错误状态检查 |
| DOM空值访问 | 所有DOM操作前检查元素是否存在 |
| 端口残留连接 | 检查端口占用，必要时更换端口 |

---

## 注意事项

1. **只修改 `tests/e2e/mobile/` 目录**，不修改其他目录
2. **测试必须是真实的移动Web测试**，不允许用API
3. **每个步骤都要检查移动端适配**
4. **测试要能看到浏览器窗口**（headed模式）
