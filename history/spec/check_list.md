# 乐高故事书籍功能 - 检查清单

本文档定义了项目开发过程中每个阶段必须检查的所有项目，确保代码质量和功能正确性。

---

## 一、测试执行检查清单

### 1.1 测试执行原则检查
- [ ] 每个阶段代码生成后立即执行测试
- [ ] 测试顺序正确：单元测试 → 功能测试 → API测试
- [ ] 测试断言验证内容符合预期，不只是函数调用断言
- [ ] 测试文件导入源代码，未重新定义函数
- [ ] 所有测试 100% 通过，无 skip 的测试用例

### 1.2 测试环境检查
- [ ] 单元测试使用 Mock 隔离外部依赖
- [ ] API测试连接本地 SQLite 数据库
- [ ] 联调测试使用本地 SQLite 数据库
- [ ] E2E测试环境与线上一致

### 1.3 覆盖率检查
- [ ] 执行 `npm run test:coverage` 检查覆盖率
- [ ] 语句覆盖率 >= 90%
- [ ] 函数覆盖率 >= 90%
- [ ] 分支覆盖率 >= 90%
- [ ] 行覆盖率 >= 95%（必须要求）
- [ ] 覆盖率报告生成在 `coverage/` 目录
- [ ] 覆盖率不足时补充测试用例

- [ ] 无类型错误（隐式类型转换）

## 三、部署规范检查清单

### 3.1 Cloudflare Pages 规范检查
- [ ] 项目结构符合 Pages 要求（静态文件在根目录或 public 目录）
- [ ] functions 目录结构正确（functions/api/*.js）
- [ ] 无 Workers 特有 API 使用（如 addEventListener）
- [ ] 无服务端渲染框架使用（如 Next.js SSR）
- [ ] 静态资源路径正确（相对路径或绝对路径）

### 3.2 Page Functions 规范检查
- [ ] 函数导出格式正确（export async function onRequest*）
- [ ] 使用标准 Request/Response 对象
- [ ] 使用 context.env 获取环境变量
- [ ] 使用 context.request 获取请求对象
- [ ] 无 Node.js 特有模块使用（如 fs, path）
- [ ] 无动态路由参数（如 /api/:id）

### 3.3 D1 数据库规范检查
- [ ] SQL 查询使用参数化（防止 SQL 注入）
- [ ] 使用 context.env.DB 执行查询
- [ ] 无本地文件数据库操作
- [ ] 无其他数据库连接（如 MySQL, PostgreSQL）
- [ ] 数据库操作在 Page Function 中执行

### 3.4 静态路由规范检查
- [ ] 所有页面为静态 HTML 文件
- [ ] 无服务端路由重写
- [ ] 无动态路由参数
- [ ] URL 参数通过查询字符串传递

### 3.5 环境变量规范检查
- [ ] 敏感信息通过环境变量配置
- [ ] 环境变量在 wrangler.toml 中声明
- [ ] 无硬编码的 API Key
- [ ] 无硬编码的数据库连接信息

## 检查执行说明

每个阶段完成后，按照以下顺序执行检查：

1. **测试执行检查** → 确保所有测试通过
2. **静态代码检查** → 确保代码质量
3. **部署规范检查** → 确保符合部署要求
4. **接口一致性检查** → 确保前后端一致
5. **Bug 记录检查** → 确保问题已记录
6. **阶段反思检查** → 确保反思完成
7. **需求反思检查** → 确保需求完整
8. **测试需求文档检查** → 确保测试覆盖

**所有检查项必须全部通过才能进入下一阶段！**
