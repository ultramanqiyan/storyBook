# 端到端数据库CRUD验收规格文档

## 一、验收目标

作为验收工程师，通过网页页面操作验证乐高故事书网站的增删改查功能，并验证数据库内容正确性。

## 二、验收环境

- 本地开发服务器: http://127.0.0.1:8788
- 数据库: SQLite (Cloudflare D1 本地模拟)
- 验证方式: 网页操作 + 数据库查询

## 三、验收范围

### 3.1 用户模块CRUD
- **Create**: 通过登录页面创建用户
- **Read**: 验证用户信息显示正确
- **Update**: 更新用户时间限制设置
- **验证**: 查询数据库确认数据正确

### 3.2 人仔模块CRUD
- **Read**: 查看预设人仔列表
- **Create**: 创建新人仔
- **Update**: 更新人仔信息
- **Delete**: 删除自定义人仔
- **验证**: 查询数据库确认数据正确

### 3.3 书籍模块CRUD
- **Create**: 创建新书籍
- **Read**: 查看书架列表和书籍详情
- **Update**: 更新书籍标题
- **Delete**: 归档书籍
- **验证**: 查询数据库确认数据正确

### 3.4 章节模块CRUD
- **Create**: 创建新章节
- **Read**: 查看章节内容
- **Delete**: 删除章节
- **验证**: 查询数据库确认数据正确

## 四、验收标准

1. 每个操作后数据库内容正确
2. 页面显示与数据库数据一致
3. 操作结果符合预期
4. 无数据丢失或错误

## 五、验收流程

1. 清空测试数据
2. 执行创建操作 → 验证数据库
3. 执行读取操作 → 验证显示正确
4. 执行更新操作 → 验证数据库
5. 执行删除操作 → 验证数据库
6. 编写详细验收报告
