# LEGO Story 项目规则

## 服务启动规则

**重要规则**：测试和诊断时，不要在运行前端服务的终端执行其他命令，否则会终止前端服务。

### 问题原因

1. **终端复用冲突**：当在运行前端服务的终端执行其他命令时，前端服务会被终止
2. **终端数量限制**：最多5个终端，测试/诊断脚本会占用终端资源

### 解决方案

使用 `start-services.bat` 脚本来同时启动两个服务：

```bash
# 在项目根目录运行
start-services.bat
```

该脚本会在独立的CMD窗口中启动服务，不会相互干扰。

### 服务地址

- 后端API服务: http://localhost:8788
- 前端Web服务: http://localhost:8082

### 测试规则

1. 确保两个服务都运行后再进行测试
2. 使用 Playwright 进行 E2E 测试
3. 不要在运行服务的终端执行其他命令
4. 如果需要重新启动服务，先停止旧服务再启动新服务

## E2E测试规则

**重要规则**：批量测试时，如果有一个case失败，立即停止测试，先修复该case，再重新运行批量测试。

### 测试流程

1. **单case测试优先**：遇到失败的case，创建单独的测试脚本验证该case
2. **定位问题**：获取崩溃日志，分析具体错误原因
3. **修复问题**：修改代码并重新构建APK
4. **验证修复**：单独运行该case确认修复成功
5. **批量测试**：修复后再运行完整的批量测试

### 效率原则

- 不要在已知问题未修复的情况下继续跑批量测试
- 每次只修复一个问题，避免同时修改多处
- 修复后立即验证，确保问题真正解决
