# APP UI自动化测试规范

## 项目概述

基于需求追溯矩阵，对lego-mobile应用进行全面的UI自动化测试，确保每个功能都得到验证，不遗漏任何功能点。

---

## 测试环境

### 运行环境
- **前端地址**: http://localhost:8081
- **后端地址**: http://localhost:8788
- **测试框架**: 手动UI测试 + 自动化验证
- **测试设备**: Web浏览器（移动端模拟）

### 测试账号
- 用户ID: 测试用户
- 测试数据: 需要预先创建测试书籍和角色

---

## 测试范围

基于需求追溯矩阵，共 **55个功能点** 需要测试：

| 模块 | 功能数 | 优先级 |
|------|--------|--------|
| 故事导演页 | 12 | P0 |
| 角色管理页 | 10 | P0 |
| 书籍详情页 | 16 | P0 |
| 章节阅读页 | 15 | P0 |
| 创建故事页 | 9 | P1 |
| 书架页面 | 6 | P1 |
| 冒险模式页 | 6 | P2 |
| 家长控制页 | 5 | P2 |
| 首页 | 4 | P1 |
