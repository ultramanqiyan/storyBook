## 四、多路径联动测试场景

### 场景1: 完整故事创作流程
```
登录 → 首页 → 创建新故事 → 选择角色 → 生成第一章 → 阅读章节 → 返回书籍详情
```

### 场景2: 角色管理流程
```
登录 → 角色页 → 创建角色 → 编辑角色 → 创建故事使用该角色 → 验证角色显示
```

### 场景3: 主题设置流程
```
登录 → 设置 → 主题设置 → 切换风格 → 保存 → 返回 → 验证设置生效
```

### 场景4: 书籍管理流程
```
登录 → 书架 → 创建书籍 → 添加角色 → 生成章节 → 编辑书籍 → 删除书籍
```

### 场景5: 家长控制流程
```
登录 → 设置 → 家长控制 → 设置时间限制 → 验证时间统计
```

## 六、数据验证要求

### 6.1 API响应验证

每个API调用后验证：
- 响应状态码为200
- 响应数据结构正确
- 数据字段完整
- 数据类型正确

### 6.2 数据更新验证

每次CRUD操作后：
- 重新查询数据验证更新
- 检查关联数据一致性
- 验证删除后数据不存在

### 6.3 跨页面数据一致性

- 创建角色后，角色页和故事创建页数据一致
- 创建书籍后，书架和首页数据一致
- 生成章节后，书籍详情和阅读页数据一致
