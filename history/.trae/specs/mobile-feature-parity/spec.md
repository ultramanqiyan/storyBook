# 移动端功能完善规范

## 项目背景

通过对比网页端和移动端的功能实现，发现移动端存在大量功能缺失。本规范旨在补齐移动端的功能差距，确保移动端与网页端功能对等。

---

## 功能对比分析

### 1. 故事导演页 (StoryDirectorScreen)

| 功能 | 网页端 | 移动端 | 差距 |
|------|--------|--------|------|
| 角色选择 | ✅ 支持选择主角(1个)、反派(可选)、配角(最多2个) | ⚠️ 只能选择角色，无角色类型设置 | **需要改进** |
| 角色类型设置 | ✅ 可设置主角/反派/配角/路人 | ❌ 缺失 | **需要添加** |
| 舞台预览 | ✅ 实时显示已选择的角色和情节元素 | ✅ 已有 | 一致 |
| 天气特效 | ✅ 多种天气视觉效果 | ✅ 已升级 | 一致 |
| 情节选择 | ✅ 天气、冒险类型、地形、装备 | ✅ 已有 | 一致 |
| 随机选择 | ✅ 一键随机选择所有情节 | ✅ 已有 | 一致 |
| 生成章节 | ✅ 满足条件后生成 | ✅ 已有 | 一致 |

**改进点：**
- 添加角色类型选择功能（主角/反派/配角/路人）
- 限制主角只能选1个、配角最多2个
- 添加反派的特殊标记

## 优先级分类

### P0 - 核心功能缺失（必须实现）

1. **故事导演页角色类型设置** - 网页端核心功能
2. **角色管理页性格和说话风格字段** - 角色创建完整性
3. **章节阅读页关键词高亮** - 阅读体验核心
4. **书籍详情页角色编辑功能** - 角色管理完整性

### P1 - 重要功能缺失

5. **章节阅读页故事创作提示面板**
6. **章节导航（上一章/下一章）**
7. **书籍详情页统计信息**
8. **书籍删除和编辑功能**
9. **名称唯一性检查**

### P2 - 体验优化

10. **书架页3D卡片效果**
11. **角色卡片美化**
12. **查看提示词功能**
13. **分享功能集成**

### P3 - 锦上添花

14. **音效反馈**
15. **主题风格设置**

---

## 技术约束

1. **只修改 lego-mobile 目录下的代码**
2. **不修改网页端代码**
3. **保持与现有API的兼容性**
4. **保持现有的代码风格和组件结构**

---

## API兼容性检查

需要确认以下API是否已存在：

| API | 用途 | 状态 |
|-----|------|------|
| `booksAPI.update()` | 更新书籍信息 | 需检查 |
| `booksAPI.delete()` | 删除书籍 | 需检查 |
| `bookCharactersAPI.update()` | 更新书籍角色 | 需检查 |
| `chaptersAPI.getNavigation()` | 获取章节导航信息 | 需检查 |

## 验收标准

### 功能验收
- 所有P0功能完整实现
- 所有P1功能完整实现
- 功能与网页端对等

### 体验验收
- 动画流畅，无卡顿
- 交互响应及时
- UI风格与现有设计一致

### 兼容性验收
- iOS/Android/Web三端运行正常
- 与现有功能无冲突
