# Android APP UI端到端测试规范

## 项目概述

对lego-mobile-android应用进行全面的UI端到端测试，覆盖所有核心功能（增删改查书籍、增删改查角色、增删改查人仔）。测试在安卓模拟器上通过UI自动化操作完成，并通过后端API验证数据操作的正确性。

---

## 测试环境

### 运行环境
- **Android模拟器**: Pixel 6 API 34
- **后端地址**: http://localhost:8788
- **测试方式**: ADB + UI Automator + 手动验证
- **监控方式**: 独立进程监控APP状态和崩溃

### 测试账号
- 用户ID: ultraman
- 测试数据: 通过API预创建或清理

---

## 测试约束

1. **单case执行**: 任何一个页面测试失败或crash，立即停止，修复后再继续下一个case，不允许批量跑
2. **进程监控**: 使用独立进程监控crash和APP进程状态，如有crash或进程退出，标记case失败
3. **全自动执行**: 所有操作包括删除都不需要确认，全部全自动执行
4. **完整覆盖**: 所有页面、所有按钮、所有输入框都必须覆盖测试,测试时候不能只是点击下，必须查看实际的内容符合预期
5. **代码限制**: 所有开发和测试代码只能在lego-mobile-android目录下

## 监控和异常处理

### 进程监控
- 使用独立进程每1秒检查APP状态
- 监控指标：进程存在性、ANR、Crash日志
- 如发现异常，立即停止当前case并记录

### Crash处理
1. 捕获crash日志
2. 截图保存
3. 记录复现步骤
4. 标记case失败
5. 修复后继续

### 数据验证
- 每个增删改操作后，通过API查询验证数据状态
- 验证数据一致性
- 记录验证结果
