# Playwright E2E测试覆盖率优化 Spec

## Why
之前的端到端测试存在登录流程问题，导致21个测试失败，代码覆盖率仅3.57%，远未达到90%目标。需要修复测试并确保所有核心功能通过网页仿真测试。

## What Changes
- 修复登录流程，确保测试能正确登录后访问其他功能
- 重构测试结构，使用 `test.beforeAll()` 进行一次登录
- 补充测试用例以达到90%行覆盖率
- 所有测试必须是网页仿真点击测试，不允许API测试

## Impact
- Affected specs: 端到端测试
- Affected code: lego-mobile/tests/playwright/core-functions.spec.js

## ADDED Requirements

### Requirement: 登录状态管理
系统 SHALL 在测试开始时完成登录，并保持登录状态供后续测试使用。

#### Scenario: 成功登录并保持状态
- **WHEN** 测试开始时执行登录
- **THEN** 后续测试可以访问需要登录的功能

### Requirement: 网页仿真测试
所有测试 SHALL 通过网页仿真点击完成，不允许直接调用API。

#### Scenario: 用户登录测试
- **WHEN** 测试用户登录
- **THEN** 通过填写表单并点击按钮完成

#### Scenario: 书籍CRUD测试
- **WHEN** 测试书籍管理
- **THEN** 通过点击UI元素完成创建、查看、编辑、删除

#### Scenario: 角色CRUD测试
- **WHEN** 测试角色管理
- **THEN** 通过点击UI元素完成创建、查看、编辑、删除

### Requirement: 代码覆盖率
每个实现文件的行覆盖率 SHALL 大于90%。

#### Scenario: 覆盖率达标
- **WHEN** 测试完成
- **THEN** 每个文件行覆盖率 >= 90%

## Test Path Design

### 用户使用网站路径
1. 登录页 → 输入用户名密码 → 点击登录 → 进入首页
2. 首页 → 点击书架 → 查看书籍列表
3. 书架 → 点击创建新故事 → 填写信息 → 创建成功
4. 书籍详情 → 点击故事导演台 → 选择元素 → 生成章节
5. 书籍详情 → 点击章节 → 阅读章节内容
6. 角色页 → 点击创建角色 → 填写信息 → 创建成功
7. 角色页 → 点击编辑按钮 → 修改信息 → 保存
8. 角色页 → 点击删除按钮 → 确认删除
9. 设置 → 主题风格设置 → 选择风格 → 保存
10. 设置 → 家长控制 → 设置时间限制
