# 端到端测试规范

## 一、测试概述

本文档定义了 LEGO 故事创作 APP 的端到端测试规范，确保每次修改后功能正常运行。

---

## 二、测试环境要求

- 前端服务: http://localhost:8081
- 后端服务: http://localhost:3000
- 测试账号: 已注册的测试用户

---

## 三、核心功能测试用例

### 3.1 用户认证流程

| ID | 测试场景 | 测试步骤 | 预期结果 |
|----|---------|---------|---------|
| AUTH-01 | 用户登录 | 1. 打开APP 2. 输入用户名密码 3. 点击登录 | 登录成功，跳转到首页 |
| AUTH-02 | 登录失败处理 | 1. 输入错误密码 2. 点击登录 | 显示错误提示，停留在登录页 |
| AUTH-03 | 用户登出 | 1. 进入设置页 2. 点击退出登录 | 返回登录页 |

### 3.2 主题风格设置流程

| ID | 测试场景 | 测试步骤 | 预期结果 |
|----|---------|---------|---------|
| THEME-01 | 切换主题风格 | 1. 进入设置页 2. 点击主题风格设置 3. 选择一个风格 4. 点击保存 | 显示保存成功，自动返回设置页 |
| THEME-02 | 取消保存 | 1. 进入主题设置 2. 选择一个风格 3. 点击返回 4. 选择不保存 | 返回设置页，设置未保存 |
| THEME-03 | 持久化验证 | 1. 保存主题设置 2. 关闭APP 3. 重新打开 | 主题设置保持不变 |
| THEME-04 | 切换Tab | 1. 进入主题设置 2. 点击不同Tab | Tab切换正常，显示对应风格列表 |

### 3.3 角色管理流程

| ID | 测试场景 | 测试步骤 | 预期结果 |

| CHAR-04 | 删除自定义角色 | 1. 点击角色卡片的删除按钮 2. 确认删除 | 角色删除成功 |

## 七、自动化测试脚本

### 7.1 测试命令

```bash
# 启动后端服务
cd lego_job && npm run dev

# 启动前端服务
cd lego-mobile && npx expo start --web

# 运行测试（如果有测试框架）
npm test
```

### 7.2 手动测试检查点

1. **页面加载**: 每个页面是否能正常加载
2. **数据展示**: 数据是否正确显示
3. **交互操作**: 点击、滑动是否正常
4. **错误处理**: 错误是否有友好提示
5. **导航跳转**: 页面跳转是否正确
