# 故事生成核心功能测试计划

## 测试目标
系统性地检查和修复故事生成、展示、添加章节、解谜这几个核心功能的问题。

## 测试环境
- 本地开发服务器: http://localhost:8788/
- 使用真实API调用（非mock）

## 测试用例

### 1. 故事创建流程测试

#### 1.1 创建新书籍并生成第一章
**步骤**:
1. 访问 http://localhost:8788/login.html
2. 输入用户名登录
3. 点击"创建新故事"
4. 输入新书籍名称，点击"创建新书籍"
5. 选择故事类型（如"冒险之旅"）
6. 选择至少一个角色，设置自定义名称和角色类型
7. 点击"开始创作"

**预期结果**:
- 显示"正在生成第一章故事..."提示
- 显示"故事创建成功！"提示
- 跳转到书籍详情页
- 章节列表显示第一章

**检查点**:
- [ ] API调用是否成功
- [ ] 返回数据格式是否正确
- [ ] 章节是否保存到数据库
- [ ] 页面跳转是否正确

#### 1.2 检查API返回数据
**检查项**:
- `/api/story` 返回: `{ success: true, title, content, puzzle, prompt, message }`
- `/api/chapters` POST 返回: `{ success: true, chapterId, chapterNumber, message }`
- `/api/books?bookId=xxx` 返回: `{ success: true, book, chapters, characters }`

### 2. 书籍详情页测试

#### 2.1 章节列表展示
**步骤**:
1. 访问书籍详情页
2. 检查章节列表是否显示

**预期结果**:
- 显示章节列表
- 每个章节显示章节号、标题、谜题图标（如有）

**检查点**:
- [ ] API调用是否成功
- [ ] 数据是否正确渲染

#### 2.2 添加章节
**步骤**:
1. 点击"添加章节"按钮
2. 等待生成完成

**预期结果**:
- 显示"生成中..."状态
- 显示"章节生成成功！"提示
- 章节列表更新
- 显示提示词卡片

**检查点**:
- [ ] API调用是否成功
- [ ] 章节是否保存
- [ ] 页面是否刷新

### 3. 冒险模式测试

#### 3.1 章节内容展示
**步骤**:
1. 在书籍详情页点击章节
2. 跳转到冒险模式页面
3. 检查故事内容是否显示

**预期结果**:
- 显示章节标题
- 显示故事内容
- 关键词高亮显示

**检查点**:
- [ ] API调用是否成功
- [ ] 数据是否正确渲染
- [ ] 关键词高亮是否生效

#### 3.2 谜题展示和交互
**步骤**:
1. 检查谜题是否显示
2. 点击选项
3. 检查答题结果

**预期结果**:
- 显示谜题问题和4个选项
- 点击选项后显示正确/错误反馈
- 播放音效
- 3次错误后显示温和惩罚

**检查点**:
- [ ] 谜题数据是否存在
- [ ] 选项渲染是否正确
- [ ] 答题逻辑是否正确
- [ ] 音效是否播放

### 4. 数据流检查

#### 4.1 story-create.html 数据流
```
用户操作 -> selectedCharacters -> charactersData -> /api/story -> storyData -> /api/chapters -> 跳转
```

#### 4.2 book.html 数据流
```
页面加载 -> /api/books -> data.chapters -> 渲染章节列表
添加章节 -> /api/story -> storyData -> /api/chapters -> loadBook()
```

#### 4.3 adventure.html 数据流
```
页面加载 -> /api/chapters -> data.chapter -> 渲染故事内容
有谜题 -> data.puzzle -> showPuzzle()
```
