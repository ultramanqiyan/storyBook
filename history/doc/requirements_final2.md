乐高故事书籍功能 - 最终版需求文档

概述

1.1 项目背景
创建一个乐高小镇，有故事书籍功能，让用户可以创建、管理和阅读自己的乐高主题故事书籍。

1.2 目标用户
主要用户：儿童（6-12 岁）
次要用户：家长辅助创作

1.3 核心价值
激发儿童创造力和想象力
提供个性化的故事创作体验
培养儿童的叙事能力
通过互动解密增强学习趣味性

2.2.3 自定义人仔创建功能
用户可以创建完全自定义的角色
上传自己的图片或选择预设图片作为基础
自定义角色名称（必填，最多 20 个字符）
自定义角色描述（可选，最多 100 个字符）
选择性格类型（从预设列表选择）
选择说话方式（从预设列表选择）
使用用户上传的图片
创建后保存到用户的人仔库，可复用
支持在角色选择步骤直接选用已创建的自定义人仔
自定义人仔与预设人仔享有同等功能（可定义自定义名字、可选择角色类型、可用于故事生成）

2.2.4 创建人仔流程
点击"创建人仔"按钮，弹出创建弹窗
上传照片（支持 JPG/PNG 格式）
点击"生成乐高人仔"按钮
输入人仔名称、性格特点、说话方式
点击"创建"保存人仔

2.2.5 创建人仔弹窗设计
复用梦想蓝图页面的样式和布局
左侧：图片上传区域（第一步）
右侧：生成结果预览区域（第二步）
底部：人仔属性输入（第三步）
保持乐高积木风格的一致性
性格风格和说话风格使用下拉选择框，选项与预设人仔保持一致

2.2.6 人仔属性
人仔 ID (character_id)
人仔名称
人仔图片内容
人设描述
性格类型
说话方式
创建者 user_id（预设人仔为 system）
创建时间
更新时间

2.2.7 图片存储规则
预设人仔图片：存储到 Cloudflar D1数据库，base64
自定义人仔图片：用户上传的图片，存储到 Cloudflare D1数据库，base64
数据库只存储图片内容 base64格式

2.2.8 自定义人仔删除处理
- 删除自定义人仔前，检查是否被书籍角色引用
- 如已被引用，提示"该人仔已在故事中使用，删除后角色将显示为默认形象"
- 用户确认后，书籍中的角色保留，人仔图片替换为默认灰色人仔
- 人仔图片从 D1 存储中删除，释放存储空间

2.3 故事角色分配

2.3.1 角色类型
主人公（必须有且仅有一个）
配角（可以有多个）
反派（可选）
路人（可选）

2.3.2 角色自定义名称
为每个选中的人仔分配角色
必须为每个选中的人仔定义一个自定义名称/昵称
自定义名称用于故事中显示，增加代入感
自定义名称长度限制：1-20 个字符
自定义名称在当前书籍中唯一，不能重复
实时保存：修改后自动保存，无需额外确认按钮
生成故事时，角色的自定义名字会被加入到 AI 提示词中
AI 生成的故事内容使用角色的自定义名字，而非原始人仔名称

2.3.3 角色选择步骤设计
预设人仔和自定义人仔均显示乐高风格图片
书籍角色自动勾选：选择书籍后，自动加载该书籍已创建的角色并默认勾选
已选角色卡片显示：
  人仔图片（16x16 大小）
  人仔名称（原始名称）
  角色类型选择
  自定义名称输入框（必填）
  性格风格标签
  说话风格标签
主角角色保持必选状态，用户可取消勾选配角或添加新角色

2.3.4 书籍角色管理
每本书籍有独立的角色列表
角色列表包含：人仔 ID、自定义名称、角色类型
续写故事时，自动加载书籍已有角色
可以在书籍详情页管理角色：
  查看所有已定义的角色（含自定义人仔）
  修改角色自定义名称
  删除不再使用的角色
  添加新角色（支持预设和自定义）

2.3.5 角色保存功能
选择角色并定义名称后，自动保存到书籍角色列表
下次创作时可直接选择已保存的角色
已保存的角色显示在"我的角色"区域
支持快速选择常用角色组合

2.3.6 自定义名称唯一性规则
- 自定义名称在当前书籍内必须唯一
- 不同书籍之间允许使用相同的自定义名称
- 同一人仔在不同书籍中可设置不同的自定义名称
- 系统不检查跨书籍的名称重复

2.3.7 角色删除历史数据处理
- 删除书籍角色时，检查是否已被章节引用
- 如已被引用，提示"该角色已在故事中使用，删除后可能影响阅读体验"
- 用户确认后，章节中的角色名称保留，但标记为"已删除角色"
- 未使用的角色可直接删除，无提示

2.3.8 主角更换历史数据处理
- 更换主角后，新章节使用新主角的自定义名称
- 已生成章节中的主角名称保持不变，确保故事连贯性
- 书籍角色列表中标注当前主角
- 用户可查看主角更换历史记录

2.5 故事生成

2.5.1 生成规则
根据选中的人仔和情节生成故事
调用火山引擎 Doubao API 生成故事文本
故事长度：100-200 字
故事风格：适合儿童阅读、轻松有趣、充满想象力
谜题生成概率：50% 的章节随机生成谜题

2.5.2 章节生成
生成故事时，同时生成章节名称
章节名称从生成的故事内容中提取关键元素
章节名称格式：简短有力，4-10 个字
AI 返回格式（无谜题）：
{
  "title": "章节名称",
  "content": "故事内容",
  "puzzle": null
}

AI 返回格式（有谜题）：
{
  "title": "章节名称",
  "content": "故事内容（在关键节点停止，等待解密）",
  "puzzle": {
    "question": "谜题问题",
    "options": ["A. 选项 1", "B. 选项 2", "C. 选项 3", "D. 选项 4"],
    "answer": "B",
    "hint": "提示内容",
    "type": "pattern"
  }
}

2.5.4 故事连贯性优化
生成新章节时，必须考虑之前所有章节的内容
提示词包含：
  书籍中所有已定义角色的信息（名称、性格、说话方式）
  前一章的故事概要
  当前选择的情节
  当前选择的角色及其自定义名称
  解谜类型提示词（随机选择一种）

2.5.5 文本生成
调用火山引擎 API：https://ark.cn-beijing.volces.com/api/v3/chat/completions
模型：doubao-1-5-pro-32k-250115
如果 API 调用失败，返回错误信息
API_KEY：通过环境变量 DOUBAO_API_KEY 配置

展示样式：
  故事内容文字保持正常大小
  人仔图片放大显示（比文字大 2 倍），位于人仔名称旁边
  关键文字高亮显示，不超过 4 个字
关键词高亮规则：
  关键人物：红色高亮（角色自定义名称）
  关键动作：紫色高亮（飞向、跳跃、奔跑、战斗、探索、发现、拯救、追逐、攀爬、游泳、飞翔、旋转、冲刺、躲闪、攻击、防御、寻找、收集、建造、修复）
  情感词：绿色高亮（开心、快乐、勇敢、害怕、兴奋、紧张、感动、惊讶、愤怒、悲伤、期待、满足、自豪、担心、安心、激动、欣慰、坚定、犹豫）
  地点词：黄色高亮（城堡、森林、太空、海底、沙漠、雪山、火山、洞穴、城市、村庄、花园、岛屿、山脉、河流、星空、云层、迷宫、宝藏、遗迹、基地）
高亮样式：彩色背景和圆角边框、字体加粗
适用场景：故事生成完成后的展示、书籍详情页的章节阅读、分享页面的故事展示

故事生成完成后，在故事内容下方展示提交给 AI 的提示词
提示词展示区域可折叠/展开
提示词包含：书籍角色信息（名称、性格、说话方式）、前情提要（如有）、本章角色、本章情节
帮助用户理解故事是如何生成的，增加透明度和教育意义

2.6 互动解密功能（选择题形式）

2.6.1 功能概述
故事生成时，在关键节点自动生成谜题及选择题选项，用户通过点击选项进行互动解密。不支持文字输入答题。

2.6.2 谜题生成规则
触发时机：故事关键节点（如遇到大门、遇到 NPC、发现宝箱）
谜题类型：系统随机选择三种模式之一
选项数量：固定 4 个选项（A/B/C/D），必须为选择题形式
选项长度：控制在 1-5 个字以内
隐藏答案：后台记录正确答案的字母（如 Flag: B），不显示给用户
答题方式：用户只能点击 A/B/C/D 选项按钮，无文字输入框

2.6.3 三种解密玩法（均为选择题）

模式一：图形规律模式（找规律）
场景：遇到一排彩色积木，缺了一块
题目示例：红、黄、蓝、红、黄、__，下一个应该是什么颜色？
选项（选择题）：
  A. 红色
  B. 黄色
  C. 蓝色
  D. 绿色
Flag：C
用户操作：点击 C 选项按钮

模式二：简单计算模式（应用题）
场景：背包里有积木，需要计算总数
题目示例：你有 5 块积木，又拿了 3 块，现在有几块？
选项（选择题）：
  A. 2 块
  B. 3 块
  C. 4 块
  D. 8 块
Flag：D
用户操作：点击 D 选项按钮

模式三：生活常识/找不同模式
场景：遇到一堆物品，需要找出不同的那个
题目示例：书本、铅笔、苹果、橡皮，哪个不一样？
选项（选择题）：
  A. 书本
  B. 铅笔
  C. 苹果
  D. 橡皮
Flag：C
用户操作：点击 C 选项按钮

2.6.4 互动解密流程
阅读：用户阅读 AI 生成的纯文本故事
遇阻：故事在关键节点停止，出现选择题（4 个选项按钮）
判定：用户点击选项，系统比对用户选择与 Flag
继续：正确则解密成功，错误则再试一次（最多 3 次）

2.6.5 答题判定逻辑（选择题专用）
用户点击选项按钮后，系统立即验证答案
比对用户点击的选项字母（A/B/C/D）与后台 Flag 是否一致
无需文字匹配，直接比对选项字母
选项按钮点击后有视觉反馈（高亮/动画）

2.6.6 提示机制
答错 2 次后，AI 主动给出提示
提示内容与题目类型相关
提示以文字形式显示在选择题下方

2.6.7 尝试次数控制
至少 3 次机会
第 3 次错误后触发温和惩罚剧情，不直接卡关
温和惩罚示例："门没打开，再想想办法"
尝试次数在选择题界面显示（如"剩余机会：2"）

2.6.8 选择题界面设计
4 个选项按钮横向或 2x2 网格排列
按钮使用乐高积木风格
点击后有动画反馈（放大/颜色变化）
正确答案：绿色高亮 + 欢快音效
错误答案：红色闪烁 + 提示文字
无文字输入框，只允许点击选项

2.7 剧情分支系统

2.7.1 选择正确
播放"答对了"提示
解锁下一章
可开启隐藏剧情

2.7.2 选择错误
播放"错了"提示
触发温和惩罚剧情

2.8 难度控制标准（三年级适配）
