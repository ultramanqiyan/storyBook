# Bug反思报告

## 遇到的Bug总结

### Bug 1: 字段名不一致
**问题描述**：API返回的字段名与前端代码使用的字段名不一致
- API返回 `original_name`
- 前端使用 `character_name`

**根本原因**：
1. 没有统一的数据格式定义
2. 两个API返回相同含义的字段使用了不同的名称
3. 测试用例使用mock数据，无法发现真实API返回的字段名差异

**修复方案**：
1. 统一API返回的字段名为 `original_name`
2. 前端添加兼容性处理：`c.original_name || c.character_name || '未知'`

**预防措施**：
1. 使用JSDoc或TypeScript定义数据类型
2. 添加数据契约测试，验证API返回的数据格式
3. 代码审查时检查字段名是否一致

---

### Bug 2: 空指针错误
**问题描述**：`Cannot read properties of null (reading 'match')`

**根本原因**：
1. 当章节列表中有"还没有章节"提示时，它没有`onclick`属性
2. `getAttribute('onclick')`返回`null`
3. 直接调用`null.match()`导致错误

**修复方案**：
```javascript
// 修复前
var lastChapterId = chapterItems[chapterItems.length - 1].getAttribute('onclick').match(/'([^']+)'/);

// 修复后
var lastChapterItem = chapterItems[chapterItems.length - 1];
var onclickAttr = lastChapterItem.getAttribute('onclick');
if (onclickAttr) {
  var lastChapterId = onclickAttr.match(/'([^']+)'/);
  // ...
}
```

**预防措施**：
1. 在访问对象属性或调用方法前，先检查对象是否存在
2. 使用可选链操作符 `?.`（如果支持）
3. 添加边界条件测试

---

### Bug 3: 变量未定义
**问题描述**：`chapterCount` 变量在函数内部未定义

**根本原因**：
1. 重构代码时，将变量从外部作用域移到了函数内部
2. 但忘记在函数内部重新定义这个变量

**修复方案**：
```javascript
// 修复前
chapter: (chapterCount + 1)  // chapterCount未定义

// 修复后
var currentChapterCount = chapterItems.length;
chapter: (currentChapterCount + 1)
```

**预防措施**：
1. 使用ESLint检查未定义的变量
2. 代码审查时检查变量作用域
3. 编写测试用例覆盖各种场景

---

### Bug 4: 谜题答题结果未传递
**问题描述**：用户答错谜题后，生成下一章时提示词仍然显示"主角成功解开了谜题"

**根本原因**：
1. 生成新章节时没有获取用户的答题记录
2. 提示词中固定显示"主角成功解开了谜题"

**修复方案**：
1. 修改chapters API，返回用户的答题记录
2. 前端获取上一章谜题时传递userId参数
3. 根据答题结果设置不同的提示词

**预防措施**：
1. 完善需求分析，考虑所有业务场景
2. 添加端到端测试，验证完整业务流程
3. 代码审查时检查业务逻辑是否完整
