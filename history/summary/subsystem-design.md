## 一、苏格拉底式提问与回答（30问）

### 1.1 关于用户子系统（5问）

**Q1：用户数据模型包含哪些字段？**

**回答**：用户数据模型（users表）：

| 字段名 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| user_id | TEXT | 是 | 用户ID，格式id_xxx_xxx |
| username | TEXT | 是 | 用户名，最多20字符 |
| email | TEXT | 否 | 邮箱（未使用） |
| avatar | TEXT | 否 | 头像（未使用） |
| parent_id | TEXT | 否 | 家长ID（未使用） |
| daily_time_limit | INTEGER | 否 | 每日时间限制，默认120分钟 |
| time_used_today | INTEGER | 否 | 今日已用时间，默认0 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

**Q2：用户ID是如何生成的？**

**回答**：用户ID通过generateId()函数生成：

```javascript
export function generateId() {
  return 'id_' + Date.now().toString(36) + '_' + Math.random().toString(36).substr(2, 9);
}
```

**生成规则**：
- 前缀：`id_`
- 时间戳部分：当前时间戳转36进制
- 随机部分：随机数转36进制取9位
- 示例：`id_m1a2b3c_d4e5f6g7h`

**Q3：用户数据如何隔离？**

**回答**：用户数据隔离机制：

1. **书籍隔离**：查询时过滤user_id
```javascript
const books = await DB.prepare(
  'SELECT * FROM books WHERE user_id = ? AND status = ?'
).bind(userId, 'active').all();
```

2. **角色隔离**：用户只能看到自己创建的角色和系统预设角色
```javascript
const characters = await DB.prepare(
  'SELECT * FROM characters WHERE creator_id = ? OR creator_id = ?'
).bind(userId, 'system').all();
```

3. **答题记录隔离**：通过user_id关联

**Q4：用户时间限制功能是否实现？**

**回答**：部分实现：
- 数据库字段存在（daily_time_limit, time_used_today）
- 后端API支持更新这些字段
- 前端没有实现时间限制UI
- 没有实现自动计时逻辑

**状态**：功能预留，未完全实现。

**Q5：用户头像如何存储？**

**回答**：头像存储设计：
- 数据库字段：avatar（TEXT类型）
- 当前状态：未实现头像功能
- 预留方案：可存储Base64编码或图片URL

### 1.2 关于故事生成子系统（5问）

**Q6：故事生成的完整流程是什么？**

**回答**：故事生成流程：

```
1. 前端请求 → POST /api/chapters-generate
2. 验证书籍ID → 检查书籍是否存在
3. 检查限制 → 章节数量<100，角色数量>0
4. 获取角色 → 从book_characters表获取
5. 获取前章 → 如果有前一章，获取摘要
6. 构建提示词 → 组装角色、前情、要求
7. 调用豆包API → 发送请求
8. 解析响应 → 提取标题、内容、谜题
9. 保存章节 → 写入chapters表
10. 保存谜题 → 写入puzzles表（如果有）
11. 返回结果 → 章节ID、标题等
```

**Q7：提示词模板是如何设计的？**

**回答**：提示词模板结构：

```javascript
const prompt = `
你是一个专业的儿童故事作家...

【角色信息】
${characters.map(c => `- ${c.name}：${c.personality}，说话方式：${c.speaking_style}`).join('\n')}

【前情提要】
${previousChapter ? previousChapter.summary : '这是故事的开始...'}

【故事要求】
1. 故事长度：100字
2. 故事风格：乐高积木世界，充满想象力和冒险
3. 目标读者：10-12岁儿童
...

【谜题要求】
谜题类型：${puzzleType}
...
`;
```

**Q8：故事生成失败如何重试？**

**回答**：失败处理机制：

| 失败类型 | 处理方式 | 用户提示 |
|----------|----------|----------|
| API调用失败 | 返回错误 | "AI服务暂时不可用" |
| 响应格式错误 | 返回错误 | "AI返回格式错误" |
| 数据库写入失败 | 返回错误 | "保存失败" |

**前端重试**：用户可点击重新生成按钮。

**Q9：故事内容如何审核？**

**回答**：内容审核机制：

1. **提示词约束**：明确要求内容适合儿童
2. **豆包模型安全**：模型内置内容安全机制
3. **人工审核**：暂未实现

**提示词中的约束**：
```
- 内容必须健康、积极、适合儿童
- 不出现暴力、恐怖、负面内容
- 语言简单易懂，适合10-12岁儿童
```

**Q10：故事生成的时间限制是多少？**

**回答**：时间限制：

| 阶段 | 时间 | 说明 |
|------|------|------|
| API调用 | 3-10秒 | 取决于豆包服务 |
| 数据库写入 | <100ms | 本地操作 |
| 总计 | 5-15秒 | 含网络延迟 |

**前端超时设置**：建议30秒。

**Q12：系统预设角色有哪些？**

**回答**：系统预设角色（来自migrations/0002_seed_data.sql）：

| 角色名 | 性格 | 说话方式 |
|--------|------|----------|
| 勇敢骑士 | 勇敢、正义、保护弱小 | 正式、有礼貌 |
| 智慧巫师 | 智慧、神秘、喜欢解谜 | 古老、充满哲理 |

**Q13：用户自定义角色如何创建？**

**回答**：创建流程：

```bash
POST /api/characters
{
  "name": "我的角色",
  "description": "角色描述",
  "personality": "性格特点",
  "speaking_style": "说话方式",
  "creator_id": "user_id"
}
```

**限制**：
- 名称最多20字符
- 描述最多200字符
- 每用户最多创建20个角色

**Q14：角色图片如何生成？**

**回答**：角色图片生成流程：

1. 用户上传图片或使用AI生成
2. 图片转换为Base64编码
3. 存储在image_base64字段

**AI生成**：调用Seedream API
```javascript
const response = await fetch('https://ark.cn-beijing.volces.com/api/v3/images/generations', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'doubao-seedream-4-0-250828',
    prompt: `乐高风格角色：${characterName}`
  })
});
```

**Q15：角色与书籍的关联如何管理？**

**回答**：通过book_characters关联表：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| book_id | TEXT | 书籍ID |
| character_id | TEXT | 角色ID |

**操作**：
- 添加角色：POST /api/book-characters
- 移除角色：DELETE /api/book-characters
- 查询角色：GET /api/book-characters?bookId=xxx

### 1.4 关于谜题子系统（5问）

**Q16：谜题数据模型包含哪些字段？**

**回答**：谜题数据模型（puzzles表）：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| puzzle_id | TEXT | 谜题ID |
| chapter_id | TEXT | 章节ID |
| question | TEXT | 谜题问题 |
| options | TEXT | 选项（JSON数组） |
| answer | TEXT | 正确答案（A/B/C/D） |
| hint | TEXT | 提示（可选） |
| puzzle_type | TEXT | 谜题类型 |
| created_at | DATETIME | 创建时间 |

**Q17：谜题如何与章节关联？**

**回答**：关联机制：

1. **生成时关联**：AI生成谜题时自动关联到当前章节
2. **查询关联**：通过chapter_id查询谜题
```javascript
const puzzle = await DB.prepare(
  'SELECT * FROM puzzles WHERE chapter_id = ?'
).bind(chapterId).first();
```

3. **一对一关系**：每个章节最多一个谜题

**Q18：谜题答案格式是什么？**

**回答**：答案格式：

- **存储格式**：字母（A/B/C/D）
- **选项格式**：JSON数组
```json
["苹果", "香蕉", "橘子", "葡萄"]
```
- **正确答案**：A（表示第一个选项）

**验证逻辑**：
```javascript
const isCorrect = puzzle.answer === userAnswer;
```

**Q19：谜题提示如何生成？**

**回答**：提示生成机制：

1. **AI生成**：豆包模型在生成谜题时可选生成提示
2. **存储位置**：hint字段
3. **显示条件**：用户答错2次后显示

**代码实现**：
```javascript
let hint = null;
if (!isCorrect && attempts >= 2) {
  hint = puzzle.hint;
}
```

**Q20：答题记录如何保存？**

**回答**：答题记录（puzzle_records表）：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| record_id | TEXT | 记录ID |
| user_id | TEXT | 用户ID |
| puzzle_id | TEXT | 谜题ID |
| user_answer | TEXT | 用户答案 |
| is_correct | INTEGER | 是否正确（0/1） |
| attempts | INTEGER | 尝试次数 |
| answered_at | DATETIME | 答题时间 |

**用途**：统计用户答题情况，未来可扩展成就系统。

### 1.5 关于分享子系统（5问）

**Q21：分享链接如何生成？**

**回答**：分享链接生成：

```javascript
const shareCode = generateId().substring(0, 8);
const shareUrl = `https://your-domain/share.html?code=${shareCode}`;
```

**存储**：
```javascript
await DB.prepare(
  'INSERT INTO shares (share_id, book_id, user_id, share_code, created_at) VALUES (?, ?, ?, ?, ?)'
).bind(shareId, bookId, userId, shareCode, now).run();
```

**Q22：分享链接有效期是多久？**

**回答**：分享链接永久有效：
- 没有过期时间字段
- 没有删除分享功能
- 用户可以无限次访问

**Q23：分享内容包含哪些信息？**

**回答**：分享内容：

| 信息 | 说明 |
|------|------|
| 书籍标题 | 原书名 |
| 章节列表 | 所有章节标题和内容 |
| 角色信息 | 书籍关联的角色 |
| 创建者 | 创建者用户名 |

**不包含**：谜题答案（保护游戏体验）

**Q24：分享统计如何实现？**

**回答**：当前未实现分享统计。

**预留字段**：
- view_count：浏览次数
- last_viewed_at：最后浏览时间

**Q25：分享隐私如何保护？**

**回答**：隐私保护措施：

1. **随机分享码**：8位随机字符，难以猜测
2. **无敏感信息**：不显示用户ID
3. **只读访问**：访客只能查看，不能修改

### 1.6 关于语音子系统（5问）

**Q26：语音识别支持哪些语言？**

**回答**：语音识别支持：

| 语言 | 支持 | 说明 |
|------|------|------|
| 中文 | ✅ | 主要支持 |
| 英文 | ✅ | 支持 |
| 混合 | ✅ | 中英混合 |

**模型**：FunAudioLLM/SenseVoiceSmall

**Q27：语音输入时长限制是多少？**

**回答**：时长限制：

| 限制项 | 值 | 说明 |
|--------|-----|------|
| 最短 | 0.5秒 | 太短无法识别 |
| 最长 | 60秒 | API限制 |
| 推荐 | 3-10秒 | 最佳识别效果 |

**Q28：语音识别准确率如何？**

**回答**：准确率情况：

| 场景 | 准确率 | 说明 |
|------|--------|------|
| 安静环境 | 95%+ | 清晰发音 |
| 嘈杂环境 | 80-90% | 背景噪音影响 |
| 儿童声音 | 85-95% | 针对儿童优化 |

**Q29：语音识别失败如何处理？**

**回答**：失败处理：

```javascript
try {
  const result = await recognizeSpeech(audioData);
  return createSuccessResponse({ text: result.text });
} catch (error) {
  return createErrorResponse('语音识别失败，请重试', 500);
}
```

**用户提示**：显示错误信息，提供重试按钮。

**Q30：语音数据如何存储？**

**回答**：语音数据不存储：
- 音频数据即时处理，不保存
- 只保存识别后的文本结果
- 保护用户隐私
