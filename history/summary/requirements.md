## 第一部分：项目全景

### 1.1 一句话描述项目

**乐高故事书是一个让10-12岁儿童创作并阅读自己专属冒险故事的互动平台**。孩子们可以创建自己的角色人仔，系统会自动生成个性化的故事内容，每章故事还包含趣味谜题，答对才能继续下一章。

### 1.2 项目核心价值

| 价值点 | 具体说明 |
|--------|----------|
| 激发创造力 | 孩子自己定义角色名字、性格，故事围绕他们的角色展开 |
| 培养阅读兴趣 | 故事短小精悍（100-200字），配合谜题增加趣味性 |
| 安全可控 | 无密码登录、每日阅读时间限制、家长可监管 |
| 零门槛使用 | 界面简洁、操作简单、4步即可创建第一个故事 |

### 1.3 目标用户画像

**主要用户：10-12岁儿童**

| 特征 | 描述 |
|------|------|
| 阅读能力 | 能独立阅读中等复杂度的故事 |
| 操作能力 | 能熟练使用触屏设备，对电脑操作有一定基础 |
| 兴趣偏好 | 喜欢冒险、魔法、解谜类内容 |
| 注意力特点 | 注意力集中时间约15-20分钟，喜欢短小精悍的内容 |
| 社交需求 | 喜欢分享自己的创作成果 |

**次要用户：家长**

| 特征 | 描述 |
|------|------|
| 关注点 | 孩子的阅读内容是否健康、使用时间是否合理 |
| 需求 | 能够设置每日使用时间限制、查看孩子的创作内容 |
| 操作水平 | 基本的电脑/手机操作能力 |

### 1.4 项目边界

**本项目是什么**：
- 一个儿童故事创作工具
- 一个互动阅读平台
- 一个谜题游戏系统

**本项目不是什么**：
- 不是电子书阅读器（内容是AI生成的，不是预设的）
- 不是社交平台（没有好友、评论、点赞功能）
- 不是教育课程（没有学习目标、进度跟踪）

### 2.2 登录流程详解

**入口**：网站首页（login.html）

**操作步骤**：

| 步骤 | 用户操作 | 系统响应 |
|------|----------|----------|
| 1 | 打开网站 | 显示登录页面，展示4个热门预设人仔 |
| 2 | 输入用户名 | 实时显示输入字符数量（最多20字） |
| 3 | 点击"开始冒险" | 系统检查用户名是否存在 |
| 4a | 用户名已存在 | 直接登录，跳转到书架页 |
| 4b | 用户名不存在 | 创建新用户，跳转到书架页 |

**业务规则**：

| 规则名称 | 规则内容 | 设计原因 |
|----------|----------|----------|
| 无密码登录 | 只需输入用户名即可登录 | 简化儿童使用流程，降低门槛 |
| 用户名唯一 | 同一用户名只能创建一个账号 | 防止账号混乱 |
| 用户名长度限制 | 最多20个字符 | 界面显示考虑 |
| 本地存储 | 用户ID存储在浏览器本地存储 | 无需记住密码，下次自动登录 |

**异常情况处理**：

| 异常情况 | 系统处理方式 |
|----------|--------------|
| 用户名为空 | 提示"请输入用户名"，按钮不可点击 |
| 用户名过长 | 超过20字符后无法继续输入 |
| 网络错误 | 显示错误提示，提供重试按钮 |

### 2.3 创建故事流程详解

**入口**：书架页点击"创建新书籍"按钮

**完整四步流程**：

#### 第一步：创建或选择书籍

| 操作 | 说明 |
|------|------|
| 创建新书籍 | 输入书名（最多50字符），点击创建 |
| 选择已有书籍 | 从书架列表中选择一本书继续创作 |

**限制**：每个用户最多创建20本书籍

#### 第二步：选择故事类型

系统提供6种故事类型，每种类型有不同的剧情走向：

| 类型图标 | 类型名称 | 故事风格 | 适合人群 |
|----------|----------|----------|----------|
| 🗺️ | 冒险之旅 | 探索未知世界、克服困难 | 喜欢探险的孩子 |
| 🔍 | 神秘谜团 | 发现线索、解开谜题 | 喜欢推理的孩子 |
| 🤝 | 友谊考验 | 朋友互助、共同成长 | 喜欢社交的孩子 |
| 🦸 | 英雄救美 | 拯救他人、正义战胜邪恶 | 喜欢英雄故事的孩子 |
| 💎 | 寻宝探险 | 寻找宝藏、解开机关 | 喜欢解谜的孩子 |
| ✨ | 魔法奇遇 | 魔法世界、奇幻冒险 | 喜欢奇幻的孩子 |

#### 第三步：选择角色

| 操作 | 说明 |
|------|------|
| 选择预设人仔 | 从系统提供的角色中选择（骑士、巫师、公主等） |
| 选择自定义人仔 | 从自己创建的角色中选择 |
| 设置角色名称 | 为角色在故事中显示的名字（最多20字符） |
| 设置角色类型 | 主角/配角/反派/路人 |

**必须选择至少一个主角**，否则无法创建故事。

#### 第四步：确认创建

系统调用AI生成第一章故事，自动跳转到书籍详情页。

### 2.4 阅读与解谜流程详解

| 区域 | 内容 |

| 右侧 | 角色卡片，显示本章涉及的角色 |

## 第四部分：业务规则详解

### 4.1 数量限制规则

| 限制项 | 限制值 | 超出处理 | 设计原因 |
|--------|--------|----------|----------|
| 用户名长度 | 20字符 | 无法继续输入 | 界面显示空间有限 |
| 书名长度 | 50字符 | 无法继续输入 | 界面显示空间有限 |
| 角色名称长度 | 20字符 | 无法继续输入 | 故事内容显示需要 |
| 每用户书籍数 | 20本 | 提示已达上限，建议删除旧书 | 数据存储考虑 |
| 每本书章节数 | 100章 | 提示已达上限 | 故事完整性考虑 |
| 每次选择角色数 | 无限制 | - | 允许多角色故事 |
| 每日阅读时间 | 120分钟（默认） | 提示今日时间已用完 | 保护儿童视力 |

### 4.2 谜题规则详解

| 规则项 | 规则内容 |
|--------|----------|
| 答题次数 | 每道谜题最多尝试3次 |
| 提示机制 | 第2次错误后显示提示 |
| 答案格式 | 单选题，A/B/C/D四个选项 |
| 正确反馈 | 显示绿色对勾，播放成功音效，显示祝贺语 |
| 错误反馈 | 显示红色叉号，播放失败音效，显示鼓励语 |
| 全错处理 | 显示正确答案，故事继续，不影响下一章生成 |
| 谜题生成 | AI根据故事内容自动生成，保证原创性 |

### 4.3 故事生成规则

| 规则项 | 规则内容 |
|--------|----------|
| 章节长度 | 100-200字（适合儿童阅读） |
| 故事风格 | 乐高积木世界，充满想象力和冒险 |
| 内容安全 | 必须适合10-12岁儿童，无暴力恐怖内容 |
| 角色出场 | 每个角色都有出场机会和台词 |
| 剧情连贯 | 新章节需承接上一章剧情发展 |
| 谜题关联 | 谜题与故事场景相关，如遇到守门人出谜语 |

### 4.4 数据隔离规则

| 数据类型 | 隔离规则 |
|----------|----------|
| 书籍 | 用户只能查看和管理自己创建的书籍 |
| 角色 | 用户可查看预设角色，只能管理自己创建的角色 |
| 答题记录 | 只有用户自己能看到自己的答题记录 |
| 分享内容 | 通过分享链接可查看他人书籍内容（只读） |

## 第五部分：非功能需求

### 5.1 性能需求

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 页面加载时间 | < 3秒 | 从点击到内容完全显示 |
| API响应时间 | < 500毫秒 | 从请求发出到收到响应 |
| 故事生成时间 | < 15秒 | 从点击生成到内容显示 |
| 并发用户数 | 100+ | 同时在线使用不卡顿 |

### 5.2 安全需求

| 安全项 | 实现方式 |
|--------|----------|
| 数据隔离 | 用户ID过滤，确保数据不串 |
| SQL注入防护 | 使用参数化查询 |
| XSS防护 | 输入验证和输出编码 |
| API密钥保护 | 存储在环境变量中 |

### 5.3 可用性需求

| 可用性项 | 目标 |

## 附录：常见问题解答

**Q1：为什么选择无密码登录？**

考虑到目标用户是10-12岁儿童，密码管理对他们来说较为困难。无密码登录降低了使用门槛，让孩子们能够快速开始创作。同时，系统通过用户名唯一性保证账号不混乱。

**Q2：故事内容如何保证适合儿童？**

系统在AI生成故事时，会在提示词中明确要求内容必须健康、积极、适合10-12岁儿童。同时，使用的豆包大语言模型内置了内容安全机制，会自动过滤不适合的内容。

**Q3：谜题太难怎么办？**

谜题设计时考虑了儿童的能力水平，难度适中。如果答错2次，系统会显示提示帮助解题。如果3次全错，系统会显示正确答案，不会阻止故事继续。

**Q4：可以修改已生成的故事吗？**

目前不支持修改已生成的章节。这是设计决策，保持故事的连贯性和AI生成的原始性。如果对故事不满意，可以创建新书籍重新开始。

**Q5：如何保护孩子的使用时间？**

系统预设了每日120分钟的阅读时间限制（数据库字段已预留），但家长控制界面尚未开发。未来版本将提供家长设置界面，可以调整每日时间限制。
