## 二、历史Bug详细分析

### 2.1 登录逻辑Bug

**问题描述**：每次登录都创建新用户，而不是查找已存在的用户。

**代码变更**：
```javascript
// 修复前
await DB.prepare('INSERT INTO users ...').run();

// 修复后
const existingUser = await DB.prepare(
  'SELECT user_id FROM users WHERE username = ?'
).bind(username).first();

if (existingUser) {
  return createSuccessResponse({ userId: existingUser.user_id });
}
// 然后才创建新用户
```

**教训**：需求描述时应明确边界条件，如"同一用户重复登录"。

### 2.2 DOM空值访问Bug

**问题描述**：点击"去解谜"时报错"Can't Read 属性 of Null reading style"。

**代码变更**：
```javascript
// 修复前
element.style.display = 'block';

// 修复后
if (element) {
  element.style.display = 'block';
}
```

**教训**：所有DOM操作都应进行空值检查。

### 2.3 CSS点击区域Bug

**问题描述**：按钮部分区域无法点击。

**代码变更**：
```css
/* 修复前 */
.bottom-bar { z-index: 10; }

/* 修复后 */
.bottom-bar { z-index: 100; pointer-events: auto; }
```

**教训**：CSS层级设计应有明确规范。
