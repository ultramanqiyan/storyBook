## 第一部分：系统架构

### 1.1 架构概览

乐高故事书采用**无服务器架构（Serverless Architecture）**，整个系统部署在Cloudflare的边缘网络上。这意味着：

- **没有传统服务器**：不需要购买、配置、维护服务器
- **自动扩展**：访问量增加时自动扩展，无需手动干预
- **按需付费**：只为实际使用的资源付费
- **全球分布**：内容缓存在全球300+城市的节点，用户就近访问

### 1.3 请求处理流程

**用户访问静态页面**：
```
用户输入URL → Cloudflare CDN查找缓存 → 返回缓存的HTML/CSS/JS
（响应时间：通常<100毫秒）
```

**用户调用API**：
```
用户操作 → 前端发送请求 → Cloudflare Pages Functions处理
→ 查询/写入数据库 → 调用第三方服务（如需要） → 返回结果
（响应时间：通常100-500毫秒，AI生成需5-15秒）
```

### 1.4 架构优势与限制

| 优势 | 说明 |
|------|------|
| 零运维 | 不需要管理服务器、操作系统、运行环境 |
| 自动扩展 | 请求量增加时自动扩展，无需手动配置 |
| 全球加速 | CDN缓存静态资源，用户就近访问 |
| 低成本 | 免费套餐足够小型项目使用 |
| 快速部署 | 代码提交后自动部署，几分钟上线 |

| 限制 | 说明 |
|------|------|
| 函数执行时间 | 单次请求CPU时间有限制（免费版10ms） |
| 数据库查询 | D1是边缘数据库，复杂查询性能有限 |
| 第三方依赖 | AI服务响应时间不可控 |
| 无状态设计 | 不能依赖服务器内存存储状态 |

## 第二部分：技术选型

### 2.1 前端技术选型

**选择：原生HTML + CSS + JavaScript**

| 对比项 | 原生技术 | React/Vue框架 |
|--------|----------|---------------|
| 学习曲线 | 低（基础Web知识即可） | 高（需要学习框架） |
| 包大小 | 0KB | 40-100KB+ |
| 首屏渲染 | 快（直接HTML） | 慢（需要JS加载后渲染） |
| SEO友好度 | 高（直接HTML） | 需要额外配置 |
| 维护成本 | 低（无框架版本更新） | 高（需要跟进框架更新） |

**选择原因**：
1. 项目页面数量少（约7个主要页面）
2. 交互复杂度低（主要是表单和列表操作）
3. 目标用户是儿童，性能敏感
4. 不需要组件复用和状态管理

### 2.2 后端技术选型

**选择：Cloudflare Pages Functions**

| 对比项 | Cloudflare Pages | Vercel | Netlify | 传统服务器 |
|--------|------------------|--------|---------|------------|
| 部署难度 | 极简 | 简单 | 简单 | 复杂 |
| 数据库集成 | D1原生集成 | 需要额外配置 | 需要额外配置 | 需要自己搭建 |
| 免费额度 | 高 | 中 | 中 | 无 |
| 全球分布 | 300+城市 | 较多 | 较多 | 单点 |
| 运维成本 | 零 | 低 | 低 | 高 |

**选择原因**：
1. 与Cloudflare D1数据库原生集成，无需额外配置
2. 免费额度足够小型项目使用
3. 全球CDN加速，用户访问快
4. 部署简单，代码提交自动部署

### 2.3 数据库选型

**选择：Cloudflare D1（SQLite）**

| 对比项 | D1 | PostgreSQL | MongoDB |
|--------|-----|------------|---------|
| 部署方式 | 边缘部署 | 中心化部署 | 中心化部署 |
| 免费额度 | 5GB存储 | 有限 | 有限 |
| 查询性能 | 边缘查询快 | 网络延迟 | 网络延迟 |
| SQL兼容 | SQLite语法 | 完整SQL | NoSQL |

**选择原因**：
1. 与Cloudflare Pages原生集成
2. 边缘查询，响应快
3. SQLite语法简单，易于开发
4. 免费额度足够项目使用

### 2.4 AI服务选型

**选择：豆包大语言模型**

| 对比项 | 豆包 | GPT-4 | 文心一言 | 通义千问 |
|--------|------|-------|----------|----------|
| 价格 | 低 | 高 | 中 | 中 |
| 中文能力 | 优秀 | 良好 | 优秀 | 优秀 |
| 响应速度 | 快 | 中等 | 快 | 快 |
| 内容安全 | 内置过滤 | 需要额外配置 | 内置过滤 | 内置过滤 |

**选择原因**：
1. 中文能力强，适合中文故事生成
2. 价格低，适合项目预算
3. 响应速度快，用户体验好
4. 内置内容安全机制，适合儿童内容

**具体使用的模型**：

| 用途 | 模型名称 | 说明 |
|------|----------|------|
| 故事生成 | doubao-1-5-pro-32k-250115 | 32K上下文，适合长文本生成 |
| 图片生成 | doubao-seedream-4-0-250828 | 高质量图片生成 |

### 2.5 语音服务选型

**选择：SiliconFlow语音识别**

| 对比项 | SiliconFlow | 百度语音 | 讯飞语音 |
|--------|-------------|----------|----------|
| 免费额度 | 有 | 有限 | 有限 |
| 中文识别率 | 高 | 高 | 高 |
| 支持格式 | 多种 | 有限 | 有限 |

**具体使用的模型**：FunAudioLLM/SenseVoiceSmall

## 第四部分：第三方服务集成

### 4.1 豆包AI服务

**服务提供商**：字节跳动

**使用的API**：

| API | 用途 | 调用频率 |
|-----|------|----------|
| Chat Completions | 故事生成 | 每次生成章节时调用 |
| Image Generation | 图片生成 | 用户创建角色时调用 |

**故事生成API详情**：

| 项目 | 说明 |
|------|------|
| 接口地址 | https://ark.cn-beijing.volces.com/api/v3/chat/completions |
| 请求方式 | POST |
| 模型 | doubao-1-5-pro-32k-250115 |
| 温度参数 | 0.8（增加创意性） |
| 最大Token | 2000 |

**调用时机**：
1. 用户创建新故事时，生成第一章
2. 用户点击"生成下一章"时，生成新章节

**输入内容**：
- 角色信息（名称、性格、说话风格）
- 前情提要（前一章摘要，如有）
- 用户选择的情节（天气、地形、冒险类型）
- 故事要求（长度、风格、内容安全）

**输出内容**：
- 章节标题
- 章节内容（100-200字）
- 谜题（问题、选项、答案、提示）

**成本估算**：
- 每章故事约消耗0.01-0.02元
- 月均100次生成约1-2元

### 4.2 SiliconFlow语音服务

## 第五部分：安全设计

### 5.1 数据隔离

**原则**：用户只能访问自己的数据

**实现方式**：

| 数据类型 | 隔离方式 |
|----------|----------|
| 书籍 | 查询时添加user_id过滤条件 |
| 角色 | 查询时返回系统预设+用户创建的 |
| 答题记录 | 查询时添加user_id过滤条件 |
| 分享内容 | 通过分享码访问，只读权限 |

### 5.2 SQL注入防护

**原则**：所有数据库查询使用参数化查询

**实现方式**：
- 使用D1的prepare().bind()方法
- 不拼接SQL字符串
- 所有用户输入都通过参数传递

### 5.3 XSS防护

**原则**：不信任任何用户输入

**实现方式**：
- 输入验证：限制输入长度和字符
- 输出编码：使用textContent而非innerHTML
- 内容安全策略：设置CSP头

### 5.4 API密钥保护

**原则**：密钥不暴露在代码中

**实现方式**：
- 存储在Cloudflare环境变量中
- 通过context.env访问
- 不在代码仓库中存储密钥

**配置方法**：
1. 进入Cloudflare Dashboard
2. 选择项目 > Settings > Environment variables
3. 添加变量名和值
4. 重新部署生效

### 5.5 CORS配置

**原则**：只允许必要的跨域请求

**实现方式**：
- 设置Access-Control-Allow-Origin
- 处理OPTIONS预检请求
- 限制允许的请求方法

## 附录：技术术语表

| 术语 | 解释 |
|------|------|
| Serverless | 无服务器架构，不需要管理服务器 |
| CDN | 内容分发网络，将内容缓存到全球节点 |
| Edge Computing | 边缘计算，在离用户最近的地方处理请求 |
| D1 | Cloudflare提供的边缘SQLite数据库 |
| Functions | Cloudflare提供的无服务器函数服务 |
| Token | AI模型的计费单位，约等于一个英文单词或半个中文字 |
| Base64 | 一种编码方式，将二进制数据转为文本 |
| CORS | 跨域资源共享，控制哪些域名可以访问API |
| SQL注入 | 一种攻击方式，通过输入恶意SQL语句攻击数据库 |
| XSS | 跨站脚本攻击，通过注入恶意脚本攻击用户浏览器 |
