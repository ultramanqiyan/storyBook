# 乐高故事书项目腾讯云迁移部署指南

## 文档说明

本文档面向需要将项目从Cloudflare迁移到腾讯云的运维人员和开发人员。阅读本文档后，您将了解：
- 为什么要迁移到腾讯云
- 迁移需要做哪些准备
- 详细的迁移步骤
- 迁移后如何验证

---

## 第一部分：迁移概述

### 1.1 为什么考虑迁移

| 对比项 | Cloudflare | 腾讯云 |
|--------|------------|--------|
| 国内访问速度 | 需要绕行海外节点 | 国内直连，速度快 |
| 数据存储位置 | 海外服务器 | 国内服务器 |
| 合规要求 | 可能需要备案 | 支持备案，合规性强 |
| 技术支持 | 英文为主 | 中文支持 |
| 成本 | 免费额度有限 | 有免费试用，按需付费 |

### 1.2 迁移涉及的组件

| 组件 | Cloudflare方案 | 腾讯云替代方案 |
|------|----------------|----------------|
| 前端托管 | Cloudflare Pages | 腾讯云静态网站托管 / 云开发 |
| 后端API | Pages Functions | 云函数 SCF / 云开发 |
| 数据库 | D1 (SQLite) | 云数据库 MySQL / 云开发数据库 |
| 文件存储 | 内置于Pages | 对象存储 COS |
| CDN加速 | Cloudflare CDN | 腾讯云 CDN |

### 1.3 迁移流程总览

```
第一阶段：准备工作（约1天）
├── 注册腾讯云账户
├── 开通所需服务
└── 准备迁移环境

第二阶段：数据库迁移（约半天）
├── 创建腾讯云数据库
├── 导出Cloudflare D1数据
└── 导入到腾讯云数据库

第三阶段：后端迁移（约1天）
├── 适配云函数环境
├── 修改数据库连接
└── 部署云函数

第四阶段：前端迁移（约半天）
├── 修改API地址
├── 部署静态网站
└── 配置CDN加速

第五阶段：验证切换（约半天）
├── 功能测试
├── 性能测试
└── 切换域名
```

## 第二部分：准备工作

### 2.1 注册腾讯云账户

**操作步骤**：

1. **访问腾讯云官网**
   - 打开浏览器，访问腾讯云官网
   - 点击右上角"免费注册"

2. **完成注册**
   - 选择注册方式（微信/邮箱/手机号）
   - 完成验证

3. **实名认证**
   - 登录后，进入"账号中心"
   - 点击"实名认证"
   - 选择个人认证或企业认证
   - 按提示上传身份证照片
   - 等待审核（通常几分钟内完成）

**注意事项**：
- 实名认证后才能使用大部分云服务
- 企业认证需要营业执照

### 2.2 开通所需服务

需要开通以下腾讯云服务：

#### 2.2.1 云开发（推荐方案）

**什么是云开发**：云开发是腾讯云提供的一站式后端服务，包含数据库、云函数、文件存储等，类似于Cloudflare的方案。

**开通步骤**：
1. 登录腾讯云控制台
2. 搜索"云开发"
3. 点击"新建环境"
4. 选择"按量付费"模式
5. 选择地域（建议选择离用户最近的地区）
6. 确认创建

**费用说明**：
- 有免费额度，小型项目足够使用
- 超出后按实际使用量计费

#### 2.2.2 云函数（备选方案）

**什么是云函数**：云函数是一种无服务器计算服务，可以运行后端代码，类似于Cloudflare Functions。

**开通步骤**：
1. 搜索"云函数"
2. 点击"立即使用"
3. 选择地域
4. 完成开通

#### 2.2.3 云数据库（备选方案）

**什么是云数据库**：云数据库是托管的数据库服务，支持MySQL等数据库。

**开通步骤**：
1. 搜索"云数据库"
2. 选择"MySQL"
3. 选择配置（小型项目选择基础版即可）
4. 设置root密码
5. 确认购买

### 2.3 安装必要工具

#### 2.3.1 腾讯云CLI工具

**用途**：通过命令行管理腾讯云资源

**安装步骤**：
1. 访问腾讯云CLI文档页面
2. 下载对应系统的安装包
3. 安装完成后，在命令提示符输入：tccli version
4. 如果显示版本号，说明安装成功

#### 2.3.2 云开发CLI工具

**用途**：管理云开发资源，部署云函数

**安装步骤**：
1. 在命令提示符输入：npm install -g cloudbase/cli
2. 安装完成后输入：tcb --version
3. 如果显示版本号，说明安装成功

## 第三部分：数据库迁移

### 3.1 方案一：使用云开发数据库（推荐）

**为什么推荐**：云开发数据库与Cloudflare D1类似，都是文档型数据库，迁移更简单。

#### 3.1.1 创建数据库集合

**操作步骤**：
1. 进入云开发控制台
2. 点击"数据库"
3. 创建以下集合（类似于数据库表）：

| 集合名称 | 说明 |
|----------|------|
| users | 用户信息 |
| characters | 角色人仔 |
| books | 书籍 |
| chapters | 章节 |
| puzzles | 谜题 |
| book_characters | 书籍角色关联 |
| puzzle_records | 答题记录 |
| shares | 分享信息 |

#### 3.1.2 导出Cloudflare D1数据

**操作步骤**：
1. 在命令提示符中输入：npx wrangler d1 export lego-story-db --output backup.sql
2. 这会将D1数据库导出为SQL文件

#### 3.1.3 转换数据格式

**说明**：云开发数据库使用JSON格式，需要将SQL数据转换为JSON。

**转换方法**：
1. 打开backup.sql文件
2. 将每个表的数据转换为JSON数组格式
3. 例如，users表的数据转换为：

```
[
  {
    "user_id": "id_xxx",
    "username": "小明",
    "daily_time_limit": 120,
    "time_used_today": 0,
    "created_at": "2026-02-26T10:00:00Z"
  }
]
```

#### 3.1.4 导入数据到云开发

**操作步骤**：
1. 进入云开发控制台
2. 选择对应集合
3. 点击"导入"
4. 选择JSON文件
5. 确认导入

### 3.2 方案二：使用云数据库MySQL

**适用场景**：如果数据量大或需要复杂查询

#### 3.2.1 创建MySQL实例

**操作步骤**：
1. 进入云数据库控制台
2. 点击"新建"
3. 选择MySQL 8.0版本
4. 选择基础版（适合小型项目）
5. 设置root密码
6. 确认购买

#### 3.2.2 创建数据库和表

**操作步骤**：
1. 进入MySQL实例详情页
2. 点击"登录数据库"
3. 执行建表语句（与D1相同的表结构）

#### 3.2.3 导入数据

**操作步骤**：
1. 使用数据库管理工具（如Navicat、DBeaver）
2. 连接到腾讯云MySQL
3. 导入backup.sql文件

## 第四部分：后端迁移

### 4.1 方案一：使用云开发云函数（推荐）

#### 4.1.1 创建云函数

**操作步骤**：
1. 进入云开发控制台
2. 点击"云函数"
3. 点击"新建云函数"
4. 输入函数名称（如"users"）
5. 选择运行环境：Node.js 18
6. 点击"下一步"完成创建

#### 4.1.2 适配代码修改

**需要修改的地方**：

| 修改项 | Cloudflare写法 | 云开发写法 |
|--------|----------------|------------|
| 数据库访问 | context.env.DB | cloud.database() |
| 环境变量 | context.env.XXX | process.env.XXX |
| 响应格式 | new Response() | return { ... } |

#### 4.1.3 部署云函数

**操作步骤**：
1. 在云函数编辑器中粘贴代码
2. 点击"保存并部署"
3. 等待部署完成

#### 4.1.4 配置HTTP访问

**操作步骤**：
1. 进入云函数详情页
2. 点击"触发管理"
3. 点击"创建触发器"
4. 选择"API网关触发"
5. 设置访问路径（如/api/users）
6. 保存配置

### 4.2 方案二：使用云函数SCF

#### 4.2.1 创建函数

**操作步骤**：
1. 进入云函数SCF控制台
2. 点击"新建"
3. 选择"从头开始"
4. 选择运行环境：Node.js 18
5. 选择"API网关触发器"
6. 完成创建

#### 4.2.2 配置API网关

**操作步骤**：
1. 进入API网关控制台
2. 创建API服务
3. 配置API路径和方法
4. 绑定云函数

## 第五部分：前端迁移

### 5.1 修改API地址

**需要修改的地方**：
- 找到前端代码中所有API调用
- 将Cloudflare的域名替换为腾讯云的域名

**修改示例**：
- 原来：https://xxx.pages.dev/api/users
- 改为：https://xxx.apigw.tencentcs.com/api/users

### 5.2 部署静态网站

#### 5.2.1 使用云开发静态托管

**操作步骤**：
1. 进入云开发控制台
2. 点击"静态网站托管"
3. 点击"上传文件"
4. 选择前端所有文件（HTML、CSS、JS）
5. 等待上传完成

#### 5.2.2 使用对象存储COS

**操作步骤**：
1. 进入对象存储控制台
2. 创建存储桶
3. 上传前端文件
4. 开启静态网站功能
5. 配置访问域名

### 5.3 配置CDN加速

**操作步骤**：
1. 进入CDN控制台
2. 点击"添加域名"
3. 输入您的域名
4. 选择源站类型（COS或自定义）
5. 配置缓存规则
6. 等待部署生效

## 第六部分：域名与备案

### 6.1 域名解析

**操作步骤**：
1. 进入域名解析控制台
2. 添加解析记录
3. 类型选择CNAME
4. 记录值填写腾讯云提供的域名
5. 保存配置

### 6.2 网站备案

**为什么需要备案**：根据中国法律规定，使用国内服务器的网站需要进行ICP备案。

**备案流程**：
1. 进入备案管理控制台
2. 点击"开始备案"
3. 填写网站信息
4. 上传所需材料
5. 等待审核（通常7-20天）

**备案所需材料**：
- 身份证照片
- 网站负责人照片
- 域名证书
- 网站真实性核验单

## 第八部分：常见问题

### 8.1 数据库连接失败

**可能原因**：
- 数据库密码错误
- 网络访问权限未配置
- 数据库实例未启动

**解决方案**：
1. 检查数据库连接配置
2. 检查安全组规则
3. 确认数据库实例状态

### 8.2 云函数调用超时

**可能原因**：
- 函数执行时间超过限制
- 内存不足
- 网络问题

**解决方案**：
1. 增加函数超时时间
2. 增加函数内存配置
3. 优化代码性能

### 8.3 API跨域问题

**可能原因**：
- API网关未配置CORS
- 前后端域名不一致

**解决方案**：
1. 在API网关配置CORS响应头
2. 或使用同一域名
