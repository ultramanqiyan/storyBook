# 乐高故事书项目需求反推文档

## 一、项目概述

### 1.1 产品定位

**乐高故事书**是一款面向10-12岁儿童的互动式故事创作平台，通过AI技术生成个性化故事，结合谜题互动增强阅读体验。

### 1.2 核心价值

| 价值点 | 描述 |
|--------|------|
| 个性化体验 | 用户可自定义角色名称，故事中融入个人元素 |
| 互动阅读 | 章节间穿插谜题，增强参与感 |
| AI驱动 | 故事内容由AI生成，每次体验独一无二 |
| 社交分享 | 支持分享故事给朋友 |

---

## 二、用户角色

### 2.1 目标用户

| 用户类型 | 年龄段 | 特点 |
|----------|--------|------|
| 主要用户 | 10-12岁儿童 | 喜欢乐高、喜欢故事、有一定阅读能力 |
| 次要用户 | 家长 | 关注内容安全、可能监控使用时长 |

### 2.2 用户场景

```
场景1：小明(11岁)放学后打开乐高故事书
- 输入自己的名字登录
- 选择喜欢的角色(蝙蝠侠)
- 开始一个冒险故事
- 阅读过程中解开谜题
- 完成后分享给同学

场景2：家长监控
- 查看孩子阅读记录
- 设置使用时长限制(功能预留)
```

|------|------|--------|

## 五、数据模型

### 5.1 实体关系图

```
┌─────────┐     ┌──────────────┐     ┌─────────┐
│  users  │────►│    books     │◄────│characters│
└─────────┘     └──────┬───────┘     └─────────┘
                     │
                     ▼
               ┌───────────┐
               │ chapters  │
               └─────┬─────┘
                     │
                     ▼
               ┌───────────┐
               │ puzzles   │
               └───────────┘
```

### 5.2 数据表说明

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| users | 用户表 | user_id, username, created_at |
| characters | 人仔表 | character_id, name, personality |
| books | 书籍表 | book_id, user_id, title, story_type |
| chapters | 章节表 | chapter_id, book_id, chapter_num, content |
| puzzles | 谜题表 | puzzle_id, chapter_id, type, answer |
| book_characters | 书籍角色关联表 | book_id, character_id, role_type |
| puzzle_records | 答题记录表 | user_id, puzzle_id, attempts, solved |
| shares | 分享表 | share_id, book_id, expires_at |

## 六、接口规范

### 6.1 API列表

| 接口 | 方法 | 功能 |
|------|------|------|
| /api/users | GET | 获取用户信息 |
| /api/users | POST | 创建用户 |
| /api/characters | GET | 获取人仔列表 |
| /api/books | GET/POST | 书籍CRUD |
| /api/chapters | GET/POST | 章节CRUD |
| /api/puzzle | GET/POST | 谜题获取和验证 |
| /api/share | GET/POST | 分享管理 |
| /api/generate | POST | AI图片生成 |
| /api/speech | POST | 语音识别 |
