## 二、典型Bug详解

### 2.1 登录逻辑Bug (BUG-001)

**问题描述**: 每次登录都创建新用户，而不是查找已存在的用户

**严重程度**: 高

**根本原因**: 需求理解偏差，未考虑重复登录场景

**错误代码**:
```javascript
await DB.prepare('INSERT INTO users ...').run();
```

**修复代码**:
```javascript
const existingUser = await DB.prepare(
  'SELECT user_id FROM users WHERE username = ?'
).bind(username).first();

if (existingUser) {
  return createSuccessResponse({ userId: existingUser.user_id });
}
await DB.prepare('INSERT INTO users ...').run();
```

**教训规则**:
> 需求描述时应明确边界条件，如"同一用户重复登录"场景

---

### 2.2 数据隔离Bug (BUG-002)

**问题描述**: 返回所有用户的人仔，而不是当前用户的

**严重程度**: 高

**根本原因**: 数据隔离逻辑缺失

**错误代码**:
```javascript
const result = await DB.prepare('SELECT * FROM characters').all();
```

**修复代码**:
```javascript
const result = await DB.prepare(
  'SELECT * FROM characters WHERE user_id = ?'
).bind(userId).all();
```

**教训规则**:
> 所有用户相关查询必须包含user_id过滤条件

---

### 2.3 DOM空值访问Bug (BUG-003)

**问题描述**: 点击"去解谜"时报错"Cannot read properties of null"

**严重程度**: 中

**根本原因**: 防御性编程缺失，未检查DOM元素是否存在

**错误代码**:
```javascript
element.style.display = 'block';
```

**修复代码**:
```javascript
if (element) {
  element.style.display = 'block';
}
```

**教训规则**:
> 所有DOM操作前必须进行空值检查

---

### 2.4 CSS点击区域Bug (BUG-004)

**问题描述**: 按钮部分区域无法点击

**严重程度**: 中

**根本原因**: CSS z-index层级设计不当

**错误代码**:
```css
.bottom-bar { z-index: 10; }
```

**修复代码**:
```css
.bottom-bar { 
  z-index: 100; 
  pointer-events: auto; 
}
```

**教训规则**:
> CSS层级设计应有明确规范，交互元素z-index应高于装饰元素

---

### 2.5 数据类型比较Bug (BUG-005)

**问题描述**: has_puzzle字段比较失败

**严重程度**: 中

**根本原因**: 数据库返回的是数字0/1，代码中与布尔值比较

**错误代码**:
```javascript
if (chapter.has_puzzle === true) { ... }
```

**修复代码**:
```javascript
if (chapter.has_puzzle === 1) { ... }
```

**教训规则**:
> SQLite布尔字段返回0/1，不是true/false

---

### 2.6 卡片翻转Bug (BUG-006)

**问题描述**: 人仔卡片翻转后按钮无法点击

**严重程度**: 中

**根本原因**: CSS transform导致z-index失效

**错误代码**:
```css
.card:hover {
  transform: rotateY(180deg);
}
```

**修复代码**:
```css
.card:hover {
  transform: rotateY(180deg);
  z-index: 1000;
}
```

**教训规则**:
> 使用3D transform时需要重新设置z-index

---

### 2.7 接口字段名不一致Bug (BUG-007)

**问题描述**: API返回original_name，前端使用character_name

**严重程度**: 中

**根本原因**: 前后端接口约定不明确

**错误代码**:
```javascript
const name = character.character_name;
```

**修复代码**:
```javascript
const name = character.original_name || character.character_name || '未知';
```

**教训规则**:
> 定义数据接口文档，前后端统一字段命名

---

### 2.8 数据库迁移Bug (BUG-008)

**问题描述**: 线上数据库结构与预期不符

**严重程度**: 高

**根本原因**: 迁移文件执行顺序问题

**解决方案**:
1. 检查现有数据库结构
2. 按正确顺序执行迁移文件
3. 添加迁移版本控制

**教训规则**:
> 数据库迁移文件应有版本号，按顺序执行

## 三、预防措施规则

### 3.1 需求分析规则

| 规则ID | 规则内容 |
|--------|----------|
| REQ-001 | 明确所有边界条件，如重复操作、空值情况 |
| REQ-002 | 用户相关功能必须考虑数据隔离 |
| REQ-003 | 定义完整的数据字典，统一字段命名 |

### 3.2 编码规则

| 规则ID | 规则内容 |
|--------|----------|
| CODE-001 | 所有DOM操作前检查元素是否存在 |
| CODE-002 | SQLite布尔字段使用0/1比较 |
| CODE-003 | 使用3D transform时重新设置z-index |
| CODE-004 | API返回数据使用兼容性处理 `a || b || 'default'` |
| CODE-005 | 所有SQL查询包含必要的WHERE条件 |

| 规则ID | 规则内容 |
|--------|----------|
| CSS-001 | 交互元素z-index > 100 |
| CSS-002 | 装饰元素z-index < 50 |
| CSS-003 | 固定定位元素需要pointer-events设置 |

### 3.4 测试规则

| 规则ID | 规则内容 |
