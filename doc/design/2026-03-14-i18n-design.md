# 国际化（i18n）设计方案

## 一、设计概述

| 项目 | 决策 |
|------|------|
| 国际化范围 | 全站（8个HTML + 4个JS文件） |
| 翻译字典 | 扩展现有 theme.js 中的 translations 对象 |
| 配置文件 | 多目录方案（config/en/ 和 config/zh/） |
| 数据迁移 | 编写迁移脚本，将数据库中文value转为英文key |
| 后端API | 返回消息码，前端根据语言显示 |
| 语言切换 | 导航栏右侧，显示 "EN / 中文" 按钮 |
| SEO支持 | 动态title、meta标签、hreflang标签 |

```html
<head>
  <title data-i18n="meta.title">StoryBook - Create Your Interactive Stories</title>
  <meta name="description" content="Create your own interactive stories...">
  <meta name="keywords" content="interactive story, story creator...">
  <link rel="alternate" hreflang="en" href="https://storybook.com/?lang=en">
  <link rel="alternate" hreflang="zh" href="https://storybook.com/?lang=zh">
  <link rel="alternate" hreflang="x-default" href="https://storybook.com/">

```

## 八、风险与注意事项

### 8.1 严重风险：value属性使用中文

- 用户选择"勇敢"后，数据库存储的是中文"勇敢"
- 解决方案：value统一使用英文key（如 `brave`），显示文字使用翻译

### 8.2 中等风险：后端AI提示词是中文

- 后端AI系统提示词是中文，影响生成的章节内容
- 解决方案：根据用户语言偏好传递不同语言的提示词

### 8.3 中等风险：配置文件全是中文

- `config/*.json` 文件中的选项全是中文
- 解决方案：创建多语言配置目录结构
