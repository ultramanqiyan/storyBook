# 预设书籍设计方案

## 概述

为每种书籍类型预设2本书籍，共8本预设书籍。每本书包含角色、章节和谜题数据。

## 设计目标

1. 每种书籍类型有2本预设书籍
2. 每本书10个章节，每章300字
3. 每个章节从4种情节卡牌类型各选1张
4. 章节保留选择的卡牌信息用于展示

## 书籍配置

### 总览

| 类型 | 书籍1 | 书籍2 |
|------|-------|-------|
| adventure (儿童冒险) | 小明的奇幻冒险 | 小勇的丛林探险 |
| fantasy (魔幻传说) | 魔法学院传说 | 龙之谷秘闻 |
| romance (都市言情) | 都市恋曲 | 咖啡馆的邂逅 |
| business (职场风云) | 职场风云录 | 创业路上的我们 |

### 章节配置

| 项目 | 配置 |
|------|------|
| 每书章节 | 10章 |
| 每章字数 | 300字 |
| 每书总字数 | 约3000字 |

### 章节卡牌配置

每个章节选择4张卡牌：

| 卡牌类型 | 数量 | 说明 |
|----------|------|------|
| weather (天气卡) | 1张 | 设定场景天气 |
| terrain (地形卡) | 1张 | 设定场景地点 |
| adventure (冒险卡) | 1张 | 设定故事事件 |
| equipment (装备卡) | 1张 | 设定使用道具 |

## 数据存储

### selected_cards 字段格式

章节表中的 `selected_cards` 字段存储JSON格式：

```json
{
  "weather": {"name": "晴天", "icon": "☀️", "description": "阳光明媚的好天气"},
  "terrain": {"name": "森林", "icon": "🌲", "description": "神秘的森林"},
  "adventure": {"name": "寻宝", "icon": "🗺️", "description": "寻找隐藏的宝藏"},
  "equipment": {"name": "放大镜", "icon": "🔍", "description": "观察细节的工具"}
}
```

## 实施步骤

1. 删除现有预设书籍数据
2. 创建新的迁移脚本
3. 插入8本预设书籍
4. 插入角色数据
5. 为每本书生成10个章节（选择卡牌、生成内容）
6. 插入谜题数据
