## 一、问题分析

### 1.1 当前问题

| 问题 | 现状 | 影响 |
|------|------|------|
| URL 数量过多 | 703 个 URL | 稀释爬虫资源，降低核心页面收录效率 |
| 章节页面过多 | ~600 个章节页面 | 占用大量爬虫配额，价值较低 |
| 权重分配不合理 | 所有书籍权重相同（0.8） | AI 预设书籍（SEO重点）未获得足够重视 |

## 二、优化方案

### 2.1 URL 精简策略

| 操作 | 移除数量 | 保留数量 |
|------|----------|----------|
| 移除所有章节页面 | ~600 | 0 |
| 移除中文书籍页面 | ~46 | 0 |
| 保留英文书籍页面 | - | ~46 |

### 2.2 权重分配方案

| 页面类型 | 当前权重 | 新权重 | 说明 |
|----------|----------|--------|------|
| 首页 `/` | 1.0 | **1.0** | 保持最高权重 |
| library.html | 0.9 | **0.8** | 书籍列表页，重要入口 |

## 三、技术实现要点

### 3.1 修改 `scripts/generate-sitemap.js`

1. **移除章节生成逻辑**（第 120-160 行）
2. **只保留英文书籍**：过滤掉 `-zh` 结尾的书籍
3. **按系列分配权重**：
   - `preset-ai-*` 且不含 `-zh`：权重 0.9
   - 其他 `preset-*` 且不含 `-zh`：权重 0.5

### 3.2 权重判断逻辑

```javascript
function getBookPriority(bookId) {
  if (bookId.includes('-zh')) return null;
  if (bookId.startsWith('preset-ai-')) return '0.9';
  return '0.5';
}
```
