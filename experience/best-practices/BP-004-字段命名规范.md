# 字段命名规范

## 统一字段命名

本项目的数据字段命名必须遵循以下规范，确保SQL Schema、API、前端三层一致。

## 常见错误

### ❌ 错误示例

```javascript
// 错误：使用了 character_id
const id = character.character_id;

// 错误：使用了 character_type
const type = character.character_type;
```

### ✅ 正确示例

```javascript
// 正确：使用 char_id
const id = character.char_id;

// 正确：使用 role_type
const type = character.role_type;
```

## 检查清单

创建或修改代码时，请确认：

- [ ] Characters表相关代码是否使用了 `char_id` 而非 `character_id`
- [ ] Characters表相关代码是否使用了 `role_type` 而非 `character_type`
- [ ] API返回数据是否与Schema字段名一致
- [ ] 前端接收数据是否与API返回字段名一致
- [ ] 测试用例mock数据是否与真实API返回一致
