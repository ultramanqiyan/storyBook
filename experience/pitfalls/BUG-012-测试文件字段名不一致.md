# BUG-012 测试文件字段名不一致

## 问题现象

测试文件中的mock数据使用了错误的字段名，与数据库Schema和API返回不一致。

## 根本原因

测试用例编写时未参照数据库Schema定义，导致字段命名不一致。

## 发现的问题

### 1. Characters表字段

| 错误字段 | 正确字段 | 位置 |
|----------|----------|------|
| `character_id` | `char_id` | books.test.js, books-characters.test.js |
| `character_type` | `role_type` | books-characters.test.js |

### 2. Chapters表字段

| 错误字段 | 正确字段 | 位置 |
|----------|----------|------|
| `chapter_number` | `order_num` | books-characters.test.js |

### 3. Plot_Cards表字段

| 错误字段 | 正确字段 | 位置 |
|----------|----------|------|
| `plot_card_id` | `card_id` | card-drop.spec.js |

## 解决方案

修改测试文件中的mock数据字段名：

```javascript
// 修复前
{ character_id: 'char-1', name: '小明' }
{ chapter_number: 1 }
{ plot_card_id: 'card-1' }

// 修复后
{ char_id: 'char-1', name: '小明' }
{ order_num: 1 }
{ card_id: 'card-1' }
```

## 预防措施

1. 测试用例必须参照数据库Schema定义字段名
2. 添加字段命名规范文档
3. 代码审查时检查测试数据字段名
