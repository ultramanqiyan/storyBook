# BUG-011 前端字段名与数据库不一致

## 问题现象

故事导演页面（director.html）角色卡牌点击选择功能失效，角色类型显示为空，创建章节时无法正确传递主角ID。

## 根本原因

前端代码使用了错误的字段名，与数据库Schema和API返回的字段名不一致：

| 数据库/API字段 | 前端错误使用 | 位置 |
|----------------|--------------|------|
| `char_id` | `character_id` | director.js:76, 230 |
| `role_type` | `character_type` | director.js:83 |

## 解决方案

修改 `src/frontend/js/director.js`：

```javascript
// 修复前
card.dataset.id = char.character_id;
<div class="char-type">${char.character_type || ''}</div>
protagonist_id: this.selectedCharacter.character_id

// 修复后
card.dataset.id = char.char_id;
<div class="char-type">${char.role_type || ''}</div>
protagonist_id: this.selectedCharacter.char_id
```

## 预防措施

1. 建立统一的数据字典文档
2. 使用JSDoc或TypeScript定义接口类型
3. 添加契约测试验证字段一致性
4. 代码审查时检查字段命名
