# BUG-005 数据类型比较Bug

## 问题现象

has_puzzle字段比较失败，谜题相关逻辑无法正确执行。

## 根本原因

数据库返回的是数字0/1，代码中与布尔值true/false比较。SQLite没有真正的布尔类型，存储的是0和1。

## 解决方案

```javascript
// 修复前
if (chapter.has_puzzle === true) { ... }

// 修复后
if (chapter.has_puzzle === 1) { ... }

// 或使用truthy判断
if (chapter.has_puzzle) { ... }
```

## 预防措施

1. SQLite布尔字段返回0/1，不是true/false
2. 使用truthy判断或与数字比较
3. 编写测试验证数据类型

## 相关代码

- `js/reader.js`
- `functions/api/chapters.js`

## 发现时间

2026-02-24

## 影响范围

所有使用布尔字段的逻辑

## 规则提炼

> **CODE-002**: SQLite布尔字段使用0/1比较，或使用truthy判断
