# BUG-007 接口字段名不一致Bug

## 问题现象

添加章节功能失败，控制台报错显示字段名未定义。API返回的字段名是`original_name`，前端代码使用的是`character_name`。

## 根本原因

前后端接口约定不明确。两个API返回相同含义的字段使用了不同的名称，测试用例使用mock数据无法发现真实API返回的字段名差异。

## 解决方案

```javascript
// 修复前
const name = character.character_name;

// 修复后
const name = character.original_name || character.character_name || '未知';
```

同时统一API返回的字段名：
```javascript
// API统一返回
{
  character_id: '...',
  original_name: '蝙蝠侠',
  custom_name: '小蝙蝠'
}
```

## 预防措施

1. 定义数据接口文档，前后端统一字段命名
2. 添加数据契约测试，验证API返回格式
3. 使用JSDoc或TypeScript定义数据类型
4. 测试使用真实API返回格式
