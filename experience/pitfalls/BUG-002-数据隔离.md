# BUG-002 数据隔离Bug

## 问题现象

返回所有用户的人份数据，而不是当前登录用户的人仔。用户A可以看到用户B的人仔角色。

## 根本原因

数据隔离逻辑缺失。SQL查询没有添加user_id过滤条件，导致返回了所有用户的数据。

## 解决方案

```javascript
// 修复前
const result = await DB.prepare('SELECT * FROM characters').all();

// 修复后
const result = await DB.prepare(
  'SELECT * FROM characters WHERE user_id = ?'
).bind(userId).all();
```

## 预防措施

1. 所有用户相关查询必须包含user_id过滤条件
2. 代码审查时重点检查数据隔离
3. 添加数据隔离测试用例

## 相关代码

- `functions/api/characters.js`
- `functions/api/books.js`

## 发现时间

2026-02-22

## 影响范围

所有涉及用户数据的API，存在数据泄露风险

## 规则提炼

> **CODE-005**: 所有SQL查询包含必要的WHERE条件，特别是user_id过滤
