# BUG-010-书籍创建流程超时

## 问题现象
E2E测试中，书籍创建流程相关测试全部超时失败：
- 应该能完成整个创建流程并验证数据库
- 步骤4点击返回书架按钮应该跳转到书架页
- 步骤4点击查看书籍按钮应该跳转到详情页
- 完整流程：登录->创建书籍->查看详情->验证角色数据
- 完整流程：登录->创建书籍->查看详情->验证卡牌数据
- 完整流程：登录->创建四种类型书籍->验证类型差异
- 完整流程：登录->书架->创建书籍->返回书架->验证书籍显示

## 根本原因
1. 测试在步骤3点击"下一步"后，等待步骤4变为active状态超时
2. 前端代码在步骤3点击"下一步"时调用API创建书籍
3. 如果API调用失败或返回错误，前端不会跳转到步骤4
4. 测试等待步骤4 active状态超时（60秒）

## 错误日志
```
Test timeout of 60000ms exceeded.
Error: page.waitForFunction: Test timeout of 60000ms exceeded.
at tests\e2e\app.spec.js:620:16
```

## 解决方案
1. 修改测试代码，增加更好的错误处理和等待逻辑
2. 在点击"下一步"后，先等待API响应
3. 检查是否有错误消息显示
4. 增加超时时间或使用更灵活的等待策略

## 修复代码
修改测试代码，在步骤3点击"下一步"后：
1. 先等待网络请求完成
2. 检查是否跳转到步骤4
3. 如果未跳转，检查是否有错误消息

```javascript
// 修改前
await page.click('#step3 button:has-text("下一步")');
await page.waitForFunction(() => {
  const step4 = document.querySelector('#step4');
  return step4 && step4.classList.contains('active');
}, { timeout: 15000 });

// 修改后
await page.click('#step3 button:has-text("下一步")');
await page.waitForResponse(resp => resp.url().includes('/api/books') && resp.method() === 'POST', { timeout: 30000 });
await page.waitForFunction(() => {
  const step4 = document.querySelector('#step4');
  return step4 && step4.classList.contains('active');
}, { timeout: 30000 });
```

## 预防措施
1. 对于涉及API调用的测试，先等待API响应
2. 使用更长的超时时间
3. 添加错误状态检查
