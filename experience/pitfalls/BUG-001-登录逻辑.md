# BUG-001 登录逻辑Bug

## 问题现象

每次登录都创建新用户，而不是查找已存在的用户。用户每次输入相同用户名登录，都会生成新的用户ID，导致历史数据丢失。

## 根本原因

需求理解偏差，未考虑重复登录场景。开发时只考虑了"用户登录"这个动作，没有考虑"同一用户重复登录"的边界条件。

## 解决方案

```javascript
// 修复前
await DB.prepare('INSERT INTO users ...').run();

// 修复后
const existingUser = await DB.prepare(
  'SELECT user_id FROM users WHERE username = ?'
).bind(username).first();

if (existingUser) {
  return createSuccessResponse({ userId: existingUser.user_id });
}
await DB.prepare('INSERT INTO users ...').run();
```

## 预防措施

1. 需求描述时应明确边界条件，如"同一用户重复登录"
2. 编写测试用例覆盖重复登录场景
3. 代码审查时检查边界条件处理
